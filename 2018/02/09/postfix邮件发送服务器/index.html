postfix邮件发送服务器

2018-02-09 
Postfix是用来替代Sendmail的，它的配置文件比Sendmail简单得多，配置相当容易。适合当做告警邮件发送服务器

yum remove -y sendmail
yum install -y postfix

主配置文件修改

/etc/postfix/main.cf

myhostname = mail.example.com      #改为自己想定义的域名
mydomain = example.com             #改为根域
inet_interfaces = all              #默认只监听localhost,需要改为all向外发信

relay_domains = $mydomain
myorigin = $mydomain
mydestination = $mydomain

chkconfig --add postfix
chkconfig postfix on
/etc/init.d/postfix start

测试发信

mail -s "邮件标题"  user@sohu.com < content.txt 
或
echo "邮件正文"  | mail -s "邮件标题"  user@sohu.com

注：其中user@sohu.com是收件人地址，content.txt里面是邮件正文

自定义定义发信人等配置

修改/etc/mail.rc (或/etc/nail.rc)

set from=$HOSTNAME@domain.com 
smtp=smtp.domain.comset 
smtp-auth-user=username 
smtp-auth-password=passwordset 
smtp-auth=login

说明：
from是定义发送的邮件地址
smtp是发生的外部smtp服务器的地址
smtp-auth-user是外部smtp服务器认证的用户名
smtp-auth-password是外部smtp服务器认证的用户密码
smtp-auth是邮件认证的方式

配置成功后，就可以使用了
