zabbix自定义shell实现邮件告警

2018-02-11

安装postfix

参考: Postfix邮件发送服务

注: 如果postfix安装后没有mail命令,则需要安装mailx

邮件发送测试

echo "aaa"|mail -s 'aaa' xxxx@qq.com

邮件发送Shell脚本

 vim /usr/local/zabbix/scripts/postfix_mail.sh 

#!/bin/bash

messages=`echo $3 | tr '\r\n' '\n'`
subject=`echo $2 | tr '\r\n' '\n'`
logdir="/var/lib/zabbix/scripts/alertscripts"

echo "${messages}" | mail -aFrom:zabbix@xxx.com -s "${subject}" xxx@xxx.com $1

注：发送邮件命令中  xxx@xxx.com 和 $1都是邮件地址,写两个原因是避免出错,暂无日志输出

测试Shell邮件发送脚本

/usr/local/zabbix/scripts/send_mail.sh xxxxx@qq.com 'aaa' 'ccc'

检查邮箱可以看到邮件

设置zabbix执行脚本的路径

vim /usr/local/zabbix/etc/zabbix_server.conf

AlertScriptsPath=/usr/local/zabbix/scripts   #把上面的脚本放这个目录下

给这个目录设置755执行权限和zabbix用户属组权限

配置”报警媒介类型”类型

管理 -- 报警媒介类型 -- 添加
名称: bj_Email_Shell
类型: scripts/脚本
脚本名称: postfix_mail.sh (按实际填写,否则会出现错误)
Script参数：
  {ALERT.SENDTO}
  {ALERT.SUBJECT}
  {ALERT.MESSAGE}

对用户添加报警方式

管理 -- 用户 -- 张三 -- 报警媒介 --添加
类型: 选择刚定义的bj_Email_Shell
收件人: zhangsan@mail.com
时间范围: 1-7,00:00-23:59;

测试

/etc/init.d/zabbix_agentd stop

等待邮件收取
