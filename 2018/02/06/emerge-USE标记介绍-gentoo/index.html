emerge USE标记介绍-gentoo

2018-02-06

Gentoo/Funtoo USE标记

安装软件时，避免不了一些依赖。那么就需要了解USE标记。
USE的简单理解如下：一个软件不只包含软件本身，还包括其组件，如，文档，插件，GUI支持等。USE就是用来标记是否要安装软件的同时安装这些组件。

默认USE标记

所有USE标志都声明在 USE 变量里面。为了让用户能方便地查找和选择USE标志，官方提供了一份默认的USE设定。这些设定是Gentoo用户通常都要用到的USE标志的集合。这个默认设置在make.defaults 文件中

查看当前正在使用的全部USE标记

root #emerge --info | grep ^USE

声明全局USE标记

通过在/etc/portage/make.conf里定义USE全局变量。
在这个变量里，添加你需要的额外USE标志，或者移除你不需要的USE标志。后者可通过在标记前面加个负号"-" 前缀来实现。

例如：

cat /etc/portage/make.conf

USE="$USE -ipv6"
USE="$USE -X -opengl -alsa -esd -kde -qt -gnome -gtk -gtk2"
USE="$USE acpi hal dbus"
USE="$USE -ldap snmp vim-syntax threads"

为单个软件包声明USE标记

通过在/etc/portage/package.use 目录中来声明单个软件包USE标记
如：
=dev-ruby/rdoc-4.0.1-r1 ruby_targets_ruby20
# required by dev-ruby/rdoc-4.0.1-r1[ruby_targets_ruby20]
# required by dev-lang/ruby-2.0.0_p598[rdoc]
# required by dev-ruby/racc-1.4.9[ruby_targets_ruby20]
=dev-ruby/json-1.8.0 ruby_targets_ruby20
......

当前可用的局部USE标记可以在网上或本机的/usr/portage/profiles/use.local.desc文件中查询到

声明临时USE标记

仅仅把USE变量声明成一个环境变量设定临时USE。比如，在安装 xxx 的时候不要装 jj 就声明：
USE ＝ "-jj" emerge xxx

USE标记优先级

优先级由低到高,排序如下：
(1)make.defaults 里面的USE默认设定
(2)用户在/etc/portage/make.conf里面的USE默认设定
(3)用户在 /etc/portage/package.use里面的USE默认设定
(4)用户作为环境变量的USE设定

USE标记颜色含义

emerge命令显示时，不同颜色USE标记的意义:

红色: 表示这次emerge用到的USE标记
黄色: 表示从上次更新后该标记被增加、删除或者Masked
蓝色: 前面带-表示这次emerge屏蔽掉的USE标记
绿色: 表示你本次编译添加的新USE标记，或者去掉的USE标记。

Emerge常用命令

emerge eix       #使用eix命令来查看软件包依赖

eix supervisor   #查看安装包具体全称、可用软件版本、软件来源等信息

emerge -s supervisor   #查询软件包名称、版本、包大小等信息

emerge -pv supervisor  #查看依赖包的USE标记,并根据以上提示进行添加

emerge app-admin/supervisor  #安装

emerge -C  "软件名称"  #卸载(不影响功能的)软件

emerge --version  #查看emerge版本信息
