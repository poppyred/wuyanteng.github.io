OpenStack-Pike-Horizen Web管理10

2018-01-14 
生产环境最好将Horizen安装在非控制节点，因为控制节点已经运行有了Apache服务，重启服务的话都会有影响。另外Horizen可以安装多个，Horizen的配置不需要部署数据库等等信息，它是直接通过API调用的Keystone

安装Openstack官方源

#安装OpenStack源
yum install -y centos-release-openstack-pike

升级包
yum -y upgrade

安装OpenStack客户端
yum install -y python-openstackclient

安装 openstack-selinux包实现对OpenStack服务的安全策略进行自动管理：
yum install -y openstack-selinux

安装Dashboard包

yum install openstack-dashboard

配置

vim /etc/openstack-dashboard/local_settings

配置OpenStack Host在控制节点上启用dashboard

OPENSTACK_HOST = "192.168.56.11"

配置哪些主机有权限访问仪表盘

ALLOWED_HOSTS = ['*',]

配置memcached会话存储服务

SESSION_ENGINE = 'django.contrib.sessions.backends.cache'

CACHES = {
    'default': {
         'BACKEND': 'django.core.cache.backends.memcached.MemcachedCache',
         'LOCATION': '192.168.56.11:11211',
    }
}

启用API 版本3

OPENSTACK_KEYSTONE_URL = "http://%s:5000/v3" % OPENSTACK_HOST

启用对域的支持

OPENSTACK_KEYSTONE_MULTIDOMAIN_SUPPORT = True   
注意: True是大写 

配置API版本

OPENSTACK_API_VERSIONS = {
    "identity": 3,
    "image": 2,
    "volume": 2,
}

把Default设置为默认域

OPENSTACK_KEYSTONE_DEFAULT_DOMAIN = "Default"

指定通过dashboard创建用户的默认角色

OPENSTACK_KEYSTONE_DEFAULT_ROLE = "user"

如果使用了扁平桥接网络, 需要禁用对三层网络服务的支持(否则dashboard会报错)

OPENSTACK_NEUTRON_NETWORK = {
    'enable_router': False,
    'enable_quotas': False,
    'enable_distributed_router': False,
    'enable_ha_router': False,
    'enable_lb': False,
    'enable_firewall': False,
    'enable_vpn': False,
    'enable_fip_topology_check': False,
}

配置时区

TIME_ZONE = "Asia/Shanghai"

重启服务

systemctl restart httpd.service

上面虽修改了

Web访问

http://192.168.56.12/dashboard/
域：Default
账号: admin   密码:admin
账号: demo    密码：demo

故障处理

  • CentOS7.4 重启完Apache访问Horizon出错，错误:500
解决方法：
vim /etc/httpd/conf.d/openstack-dashboard.conf

搜索：WSGISocketPrefix run/wsgi
在下面加入如下一行：WSGIApplicationGroup %{GLOBAL}

最后重启Apache
systemctl restart httpd
