docker入门-基本管理1

2018-01-22 
docker虚拟化基于内核实现，类似的开源技术还有LVS

docker解决了什么问题

1. 资源利用效率低
2. 单物理机多应用无法有效隔离(进程空间、cpu资源、磁盘)
3. 运维部署不便
4. 测试、版本管理复杂
5. 迁移成本高
6. 传统虚拟机、空间占用大，启动慢，管理复杂

docker特性

基于CGroup和namespace实现
注：官方文档要求Linux kernel至少3.8以上，且docker只能运行在64位的系统中

docker版本

docker-ce是社区版
docker-ee是企业版

获取阿里云Docker镜像源

登陆：https://cr.console.aliyun.com  ---> 镜像加速器

使用:

vim /etc/docker/daemon.json

sudo mkdir -p /etc/docker
sudo tee /etc/docker/daemon.json <<-'EOF'
{
  "registry-mirrors": ["https://of3qht19.mirror.aliyuncs.com"]
}
EOF

sudo systemctl daemon-reload
sudo systemctl restart docker

docker默认根目录: /var/lib/docker,按照如下修改docker根目录

vim /etc/docker/daemon.json

{
  "registry-mirrors": ["https://56px195b.mirror.aliyuncs.com"],
  "dns": ["192.168.56.2","223.5.5.5"],    
  "data-root": "/data/docker"

如果容器内无法上网，可以在此配置文件指定dns；

vim /usr/lib/systemd/system/docker.service

注释掉默认的ExecStart一行，添加如下行：

Docker管理命令

镜像管理

docker images    //列出本地所有镜像
docker search <image_id或name>   //查找images
docker pull  <image_id>  //下载image
docker push  <image_id>   //上传image

容器管理

docker run --rm -dit --name centos7 <IMAGE_ID> /bin/bash

参数说明:
   -i 标准输入给容器
   -t 分配一个虚拟终端
   -d 以守护进程方式运行
   -P 默认匹配docker容器的5000端口号到宿主机的<49153-65535>端口
   -p <local_PORT>:<CONTAINER_PORT>  指定端口号
   --name： 指定容器的名称
   --rm：退出时删除容器
   /bin/bash：执行bash脚本


docker stop <CONTAINER_ID>：停止container
docker start <CONTAINER_ID>：重新启动container
docker ps  列出容器
  -l：显示最后启动的容器
  -a：同时显示停止的容器，默认只显示启动状态

docker attach <CONTAINER_ID>   连接到启动的容器
docker logs <CONTAINER_ID>     输出容器日志, -f参数可以实时输出 

docker top <CONTAINER_ID>   查看容器中运行的进程
docker diff <CONTAINER_ID>  查看容器中的变化
docker inspect <CONTAINER_ID>  查看容器详细信息（输出为Json）
docker extc -it <CONTAINER> <COMMAND>：在容器里执行命令，并输出结果
docker rm -f <CONTAINER_ID>  删除容器

# 将镜像导出到文件
docker save CONTAINER_ID > update.tar    #默认当前目录

#导入镜像
docker load < update.tar

网络管理

docker run -P     //随机分配端口号

绑定特定端口号（主机的所有网络接口的5000端口均绑定容器的5000端口）
docker run -p 5000:5000

绑定主机的特定接口的端口号
docker run -p 127.0.0.1:5000:5000

绑定udp端口号
docker run -d -p 127.0.0.1:5000:5000/udp 

查看容器的5000端口对应本地机器的IP和端口号
docker port <CONTAINER_ID> 5000

使用Docker Linking连接容器
docker run -d -P --name <CONTAINER_NAME> --link <CONTAINER_NAME_TO_LINK>:<ALIAS> 
注：Docker为源容器和接收容器创建一个安全的通道，容器之间不需要暴露端口，接收的容器可以访问源容器的数据

数据管理

* 数据卷可以在容器间共享和重复使用
* 可以直接修改容器卷的数据
* 容器卷里的数据不会被包含到镜像中
* 容器卷保持到没有容器再使用它

可以在容器启动的时候添加-v参数指定容器卷 (也可以在Dockerfile里用VOLUMN命令添加)
docker run -d -P --name web -v /webapp training/webapp nginx /bin/bash

也可以将容器卷挂载到宿主机目录或宿主机的文件上，<容器目录或文件>的内容会被替换为<宿主机目录或文件>的内容，默认容器对这个目录有可读写权限
docker run -d -P --name web -v <宿主机目录>:<容器目录> nginx /bin/bash

可以通过指定ro，将权限改为只读
docker run -d -P --name web -v <宿主机目录>:<容器目录>:ro training/webapp  nginx /bin/bash

容器卷共享

首先启动了一个容器，并为这个容器增加一个数据卷/webapp，然后启动另一个容器，共享这个数据卷
docker run --name logs -v /var/log busybox true

docker run --volumes-from logs myapp
当容器myapp被删除时，容器卷也不会被删除，只有所有容器不再使用此容器卷时，才会被删除
docker rm -v  删除容器卷

退出docker容器

正确退出方式
按住Ctrl不松手，再依次按"P"和“Q”键

慎用exit
