docker高级进阶3-创建docker网络

2018-01-22

查看docker网络模式

[root@linux-node1 ~]# docker network ls
NETWORK ID          NAME                DRIVER              SCOPE
2764e6841f4b        bridge              bridge              local
71a2bf3def0a        host                host                local
18de574e6899        none                null                local

指定容器的网络模式

docker run -it --net=host nginx

创建docker网络

创建名为“my_net”的docker网络,驱动:桥接
[root@linux-node1 ~]# docker network create --driver bridge my_net
ae86a1c4fe95d544d7ca80b7996bfe0b88fda070919ca77644d0fe18cdb1b3ff

注：默认网络不能指定IP地址

运行容器并指定网络
[root@linux-node1 ~]# docker run -it --rm --network=my_net busybox
/ # 
/ # ip a
35: eth0@if36: <BROADCAST,MULTICAST,UP,LOWER_UP,M-DOWN> mtu 1500 qdisc noqueue 
    link/ether 02:42:ac:12:00:02 brd ff:ff:ff:ff:ff:ff
    inet 172.18.0.2/16 brd 172.18.255.255 scope global eth0
       valid_lft forever preferred_lft forever
/ #

同一宿主机不同网段docker互访

创建my_net2并手动指定一个子网
[root@linux-node1 ~]# docker network create --driver bridge --subnet 172.22.16.0/24 --gateway=172.22.16.1 my_net2
2c0275289202761452d0066a94e06c31f615cb5a0f8d82284c880ddb40ec54e5

运行默认网络的容器
docker run -it --rm --name gary1 busybox

再运行一个my_net2的自定义网络地址的容器
docker run -it --rm --network=my_net2 --ip 172.22.16.99 --name gary2 busybox

把gary1 connect到gary2的网络里面
docker network connect my_net2 gary1

这样就可以在上面两个容器内互相ping了
ping gary1
ping gary2

跨宿主机的容器间网络互通

* 配置跨宿主机进行容器网络互通
环境：3台虚拟机作为宿主机

虚机3启动progrium/consul容器
[root@linux-node3 ~]# docker run -d -p 8500:8500 --name consul progrium/consul -server -bootstrap
28fc3dda5acba36694fe50f8c5ccc7d40167ae76c65e3f608e5600a39b8e04e5
Web浏览器访问8500端口:  http://192.168.56.13:8500/ui/#/dc1/services
--------------------------------------------------------------------------

虚拟机2确认参数
[root@linux-node2 ~]# dockerd --help |grep cluster
      --cluster-advertise string                Address or interface
      --cluster-store string                    URL of the distributed
      --cluster-store-opt map                   Set cluster store

------------------------------------------------------------------------
虚拟机1配置-确保有如下配置：
vim /usr/lib/systemd/system/docker.service
ExecStart=/usr/bin/dockerd -H tcp://192.168.56.11 -H unix:///var/run/docker.sock

虚拟机2配置-确保有如下配置：
vim /usr/lib/systemd/system/docker.service
ExecStart=/usr/bin/dockerd -H tcp://192.168.56.12 -H unix:///var/run/docker.sock


虚拟机1进行如下配置-添加2行内容，如下(注意分号)：
[root@linux-node1 ~]# cat /etc/docker/daemon.json
{"registry-mirrors": ["http://hub-mirror.c.163.com"],
 "cluster-store":"consul://192.168.56.13:8500",
  "cluster-advertise":"192.168.56.11:2375"
}

重启服务：
[root@linux-node1 ~]# systemctl daemon-reload
[root@linux-node1 ~]# systemctl restart docker

----------------------------------------------------
虚拟机2进行如下配置-添加2行内容，如下(注意分号)：
[root@linux-node1 ~]# cat /etc/docker/daemon.json
{"registry-mirrors": ["http://hub-mirror.c.163.com"],
 "cluster-store":"consul://192.168.56.13:8500",
  "cluster-advertise":"192.168.56.12:2375"
}
[root@linux-node2 ~]# systemctl daemon-reload
[root@linux-node2 ~]# sudo systemctl restart docker

-----------------------------------------------------
最后浏览器访问：http://192.168.56.13:8500/ 
依次点击“Key/Value” -- "docker" -- "nodes",如果出现两个NODES则说明配置成功
DOCKER/NODES/ +
192.168.56.11:2375
192.168.56.12:2375

-----------------------------------------------------------------

然后在node1和node2的宿主机上查看网络
[root@linux-node1 ~]# docker network ls
NETWORK ID          NAME                DRIVER              SCOPE
fa1ccee83f54        docker_gwbridge     bridge              local
fa66fe2c974f        ov_net1             overlay             global
可以看到会多出一个docker_gwbridge,这个docker_gwbridge来确保跨机器docker不同容器之间进行通信

此时两个宿主机也可以ping正常访问外网，那么查看下docker容器的路由，这是因为这个路由走的是默认网络
/ # ip route list
default via 172.19.0.1 dev eth1 
10.0.0.0/24 dev eth0 scope link  src 10.0.0.2 
172.19.0.0/16 dev eth1 scope link  src 172.19.0.2 
可以看到默认网关是172.19.0.1，所以刚刚创建的ov_net1网络和docker_gwbridge只负责跨宿主机之间的docker容器通信，只有这一个作用。
