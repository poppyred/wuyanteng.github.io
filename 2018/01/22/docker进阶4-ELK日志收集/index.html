docker进阶4-ELK日志收集

2018-01-22

实现方法1：

通过filebeat 实现ELK收集日志
流程：nginx日志通过filebeat写入到es，最后kibana展示

node3配置

[root@linux-node3 ~]# docker run -p 5601:5601 -p 9200:9200 -p 5044:5044 -it --rm --name elk sebp/elk 
错误日志：max virtual memory areas vm.max_map_count [65530] is too low, increase to at least

解决方法 - 修改内核参数
sysctl vm.max_map_count=262144

再次启动elk容器(至少4G内存)
Web访问测试：
ELK: http://192.168.56.13:9200
kibana:http://192.168.56.13:5601/

node1/node2/node3宿主机都需要安装filebeat

wget https://artifacts.elastic.co/downloads/beats/filebeat/filebeat-6.1.1-x86_64.rpm
rpm -ivh filebeat-6.1.1-x86_64.rpm

node1上运行nginx容器
docker run -it -p 80:80 --name nginx_v2 nginx
docker ps确认容器启动


[root@linux-node1 ~]# vim /etc/filebeat/filebeat.yml   #修改filebeat
  enabled: true         #改为true
  #- /var/log/*.log      #注释源docker日志目录
  - /var/lib/docker/containers/*.*/*.log     #重新指定docker日志目录
  output.elasticsearch:    #输出到ES 192.168.56.13:9200
  # Array of hosts to connect to.
  hosts: ["192.168.56.13:9200"]

 重启filebeat

访问node1节点的nginx容器

[root@linux-node1 ~]# curl 127.0.0.1   #模拟请求来产生nginx日志

刷新访问ES查看生成的日志
http://192.168.56.13:9200/

最后访问并配置kibana,把ES数据展示到kibana
http://192.168.56.13:5601

实现方法2(基于方法1)：

node3上操作

[root@linux-node3 ~]# mkdir -p /fluentd_log  #把本地目录挂载到容器中
docker run -d -p 24224:24224 -p 24224:24224/udp -v /fluentd_log:/fluentd/log fluent/fluentd

[root@linux-node3 ~]# ss -lntup |grep 24224    #查看端口是否已经启动

更改日志驱动,参考：
[root@linux-node3 docker]# cat daemon.json 
{"registry-mirrors": ["http://hub-mirror.c.163.com"],
 "log-driver": "fluentd",
 "log-opts": {
        "fluentd-address":"192.168.56.13:24224",
        "tag":"linux-node1.example.com"
        }
}

重载配置
systemctl restart docker

node3节点安装配置filebeat

vim /etc/filebeat/filebeat.yml 
  enabled: true      #改为true
  paths:
    - /fluentd_log/*.log     #修改日志目录
    output.elasticsearch:    #输出到ES 192.168.56.13:9200
    # Array of hosts to connect to.
    hosts: ["192.168.56.13:9200"]

/etc/init.d/filebeat start
/etc/init.d/filebeat status

然后访问56.11的nginx
curl 127.0.0.1   #多次请求，此时/fluentd_log目录已经产生了日志文件

再次访问ES，再次访问配置kibana展示
