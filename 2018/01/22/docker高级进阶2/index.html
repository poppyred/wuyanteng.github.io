基础环境配置

1. hosts文件参考
192.168.56.11 linux-node1.example.com linux-node1

2、时间同步

制作自己的docker镜像

启动CentOS来制作nginx镜像
[root@linux-node1 ~]# docker run -it --name my_nginx centos bash
[root@e83627795728 /]# 
然后在容器内定制自己的服务,可以自己安装也可以yum

注：在一个docker实例中运行多实例，建议使用supervisor工具统一管理
[root@e83627795728 /]# yum install supervisor -y
vim /etc/supervisord.conf
关注如下参数：
    [include]
    files = supervisord.d/*.ini

做好之后exit退出容器

镜像制作：

提交一个新的image
[root@linux-node1 ~]# docker commit -m "LNMP1.0" aoct/lnmp2
sha256:503ff248064e6f7d98df5a3087519cfb2c0aaa0d4953db9a29be1df9af1ad47c
注：-m参数用来指定commit message

[root@linux-node1 ~]# docker images |grep aoct/lnmp2
aoct/lnmp2         503ff248064e        53 seconds ago      438MB

运行自己制作的镜像
[root@linux-node1 ~]# docker run -it -p 80:80 --name lnmp1 503ff248064e supervisord
浏览器访问:http://192.168.56.11可以看到nginx页面

使用docker运行mysql容器

docker run --name first-mysql -p 3306:3306 --rm mysql -e  MYSQL\_ROOT\_PASSWORD=123456 -d mysql

上述命令各个参数含义：
run   运行一个容器
--name 定义这个容器的名称
-p 3306:3306 表示在这个容器中使用3306端口(第二个)映射到本机的端口号也为3306(第一个)
-d  表示使用守护进程运行，即服务挂在后台
--rm参数意思是当容器停止时，将删除容器。（生产禁止使用此参数）

启动后测试mysql连接
[root@linux-node1 ~]# mysql -uroot -p123456 -P3306 -h 127.0.0.1

内存/CPU容器测试

启动一个内存容器,并分配内存50M
[root@linux-node1 ~]# docker run -it --rm -m 50M aclstack/mem mem

启动一个CPU容器,并分配工作的cpu核心
[root@linux-node1 ~]# docker run -it --cpuset-cpus 0 -it --rm -c 2048
 -c 2048  指定CPU 权重,默认1024;
 --cpuset-cpus 0  指定在哪个CPU核心上运行

关于CPU总结：
当两个docker容器同时竞争CPU资源时，权值高的占用CPU资源越高；但当一个资源权重高，但实际使用资源较低，那么权值高的容器资源占用就较低
