docker容器运行后为什么立即退出

2018-01-22

docker容器后台/交互运行

使用交互式启动容器

docker run -dit --name os centos

注：-d参数让容器后台运行
    -i标准输入给容器
    -t参数给容器分配伪终端

查看容器状态
docker ps -a

进入已存在的容器
docker attach os

退出容器时，使用[ctrl+P][ctrl+q]退出这样就不会终止容器运行
注：退出慎重使用exit

容器内的程序为什么需要前台启动？

重要：Docker容器后台运行,就必须有一个前台运行的进程.

演示说明docker机制：

启动如下centos容器来说明
docker run -dit --name os2 centos /bin/bash

进入容器
docker attach aeb60d3e59ac

查看容器内的进程
[root@aeb60d3e59ac /]# ps -aux
USER   PID %CPU %MEM    VSZ   RSS TTY  STAT START   TIME COMMAND
root   1  0.0  0.1  11780  1872 pts/0    Ss   04:01   0:00 /bin/bash

可以看到上面的第一个/bin/bash进程ID是1

这说明，当这个pid进程为1的进程退出，由于docker自身机制（认为无事情可做了）会将容器停止运行。

以nginx和fpm为例

正常情况下,我们配置启动服务只需要启动service即可,例如
service nginx start && service php5-fpm start

但是,这样做,nginx和fpm均为后台进程模式运行,就导致docker前台没有运行的应用,这样的容器,后台启动后,会立即自杀,因为他觉得他没事可做了.

解决方法:
可以写成:
service php5-fpm start && nginx -g "daemon off;"
或
service nginx start && service php5-fpm start && tail -f /var/log/nginx/error.log
