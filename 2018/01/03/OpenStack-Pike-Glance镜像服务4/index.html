OpenStack_Pike-Glance镜像服务4

2018-01-03 
Glance镜像服务：简单说需要启用2个接口或者说2个进程。

Glance-api: 用于接受云系统镜像的创建、删除、读取请求。
Glance-Registry: 用于云系统的镜像注册服务。

Glance-api功能

Glance-api在功能上与nova-api十分相似，都是接收REST API请求，然后通过其他模块（glance-registry及image store）来完成诸如镜像的查找、获取、上传、删除等操作。
api默认监听端口：9292

Glance-Registry功能

Glance-Registry用于与MySQL数据库交互，用于存储或获取镜像的元数据（metadata），提供镜像元数据相关的REST接口，通过glance-registry,可以向数据库中写入或获取镜像的各种数据，glance-registry监听端口:9191。Glance的数据库中有两张表，一张是image表，另一张是image property表。image表保存了镜像格式，大小等信息；而image property表则主要保存镜像的定制化信息。

image-store

image-store是一个存储的接口层，通过这个接口，glance可以获取镜像，image store支持的存储有Amazon的S3，OpenStack本身的swift，还有诸如ceph、sheepdog,GlusterFS等分布式存储。image store是镜像保存与获取的接口，它仅仅是一个接口层，具体的实现需要外部存储的支持。

配置数据库

CREATE DATABASE glance;
grant all privileges on glance.* to 'glance'@'%' identified by 'glance';
grant all privileges on glance.* to 'glance'@localhost identified by 'glance';

创建Glance用户并加入service项目

source /scripts/admin-openrc

创建glance用户(密码：glance)
openstack user create --domain default --password-prompt glance

添加glance用户到admin角色和service项目中
openstack role add --project service --user glance admin

创建glance服务,类型: image

openstack service create --name glance --description "OpenStack Image" image

创建image服务 API endpoint

openstack endpoint create --region RegionOne image public http://192.168.56.11:9292
openstack endpoint create --region RegionOne image internal http://192.168.56.11:9292
openstack endpoint create --region RegionOne image admin http://192.168.56.11:9292

安装和配置Glance

yum install -y openstack-glance

配置glance-api.conf 

[database]
connection = mysql+pymysql://glance:glance@192.168.56.11/glance

[keystone_authtoken]
auth_uri = http://192.168.56.11:5000
auth_url = http://192.168.56.11:35357
memcached_servers = 192.168.56.11:11211
auth_type = password
project_domain_name = default
user_domain_name = default
project_name = service
username = glance
password = glance

[paste_deploy]
flavor = keystone

配置glance存储为本地存储(生产可配置ceph或cinder)

[glance_store]
stores = file,http
default_store = file
filesystem_store_datadir = /var/lib/glance/images/

--------------------------------------------------------------
Glance镜像存储支持列表Possible values:
     * file
     * filesystem
     * http
     * https
     * swift
     * swift+http
     * swift+https
     * swift+config
     * rbd
     * sheepdog
     * cinder
     * vsphere

配置glance-registry.conf

[database]
connection = mysql+pymysql://glance:glance@192.168.56.11/glance

[keystone_authtoken]
auth_uri = http://192.168.56.11:5000
auth_url = http://192.168.56.11:35357
memcached_servers = 192.168.56.11:11211
auth_type = password
project_domain_name = default
user_domain_name = default
project_name = service
username = glance
password = glance

[paste_deploy]
flavor = keystone

同步数据库

su -s /bin/sh -c "glance-manage db_sync" glance

启动服务

systemctl enable openstack-glance-api.service openstack-glance-registry.service
systemctl start openstack-glance-api.service openstack-glance-registry.service
systemctl status openstack-glance-api.service openstack-glance-registry.service

验证端口

netstat -lntup    #查看9292和9191端口是否已经启动

Glance排错

OpenStack服务组件之间都是通过REST API来访问的，都是基于HTTP协议，所以可以根据HTTP错误请求代码来快速判断故障点并进行修复
grep "ERROR" /var/log/glance/api.log 
grep "ERROR" /var/log/glance/registry.log

镜像管理

查看空的glance镜像列表

[root@controller ~]# glance image-list
+----+------+
| ID | Name |
+----+------+
+----+------+

下载一个最小的源镜像

wget http://download.cirros-cloud.net/0.3.4/cirros-0.3.4-x86_64-disk.img

上传镜像

使用磁盘格式:QCOW2 ，容器格式:bare上传镜像到镜像服务并设置权限为public
openstack image create "cirros" --file cirros-0.3.4-x86_64-disk.img --disk-format qcow2 --container-format bare --public

注：留意返回结果中的镜像ID

确认镜像上传并验证属性

openstack image list
+--------------------------------------+--------+--------+
| ID                                   | Name   | Status |
+--------------------------------------+--------+--------+
| b0351ef2-b299-4082-ad5e-a7b1ca3353b4 | cirros | active |
+--------------------------------------+--------+--------+

注：已经active的镜像可用于创建虚拟机

备注：

由glance配置文件可以确定，glance镜像存储路径定义在了/var/lib/glance/images目录下。去这里看下
[root@localhost]# cd /var/lib/glance/images/ && ls -l
-rw-r----- 1 glance glance 13M Dec 31 22:34 b0351ef2-b299-4082-ad5e-a7b1ca3353b4
从返回结果中可以看到它是以镜像ID作为镜像名称来存储的

[root@localhost images]# file b0351ef2-b299-4082-ad5e-a7b1ca3353b4 
b0351ef2-b299-4082-ad5e-a7b1ca3353b4: QEMU QCOW Image (v2), 41126400 bytes
从上面可以看出，镜像格式为： QEMU QCOW2镜像
