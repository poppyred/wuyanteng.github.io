OpenStack历史发行版说明

OpenStack Releases版本发行说明

【控制节点】

安装OpenStack官方源

#安装OpenStack源
yum install -y centos-release-openstack-pike

升级包
yum -y upgrade

安装OpenStack客户端
yum install -y python-openstackclient

安装 openstack-selinux包实现对OpenStack服务的安全策略进行自动管理：
yum install -y openstack-selinux

编译安装MySQL5.6.36(生产环境可使用MySQL主从)

#安装依赖包
yum groupinstall "Development Tools"
yum install ncurses-devel libaio-devel -y
rpm -qa ncurses-devel libaio-devel

#安装预编译工具cmake
yum install -y cmake

#创建用户
useradd -s /sbin/nologin -M mysql
id mysql

#官方下载mysql-版本:mysql-5.6.36
wget https://cdn.mysql.com/archives/mysql-5.6/mysql-5.6.36.tar.gz
#检查md5是否与官方一致
md5sum mysql-5.6.36.tar.gz   

#解压、进入目录编译
tar -zxvf mysql-5.6.36.tar.gz && cd mysql-5.6.36

#编译
cmake . -DCMAKE_INSTALL_PREFIX=/usr/local/mysql-5.6.36 \
-DMYSQL_DATADIR=/usr/local/mysql-5.6.36/data \
-DMYSQL_UNIX_ADDR=/usr/local/mysql-5.6.36/data/mysql.sock \
-DDEFAULT_CHARSET=utf8 \
-DDEFAULT_COLLATION=utf8_general_ci \
-DWITH_EXTRA_CHARSETS=all \
-DWITH_INNOBASE_STORAGE_ENGINE=1 \
-DWITH_FEDERATED_STORAGE_ENGINE=1 \
-DWITH_BLACKHOLE_STORAGE_ENGINE=1 \
-DWITHOUT_EXAMPLE_STORAGE_ENGINE=1 \
-DWITH_ZLIB=bundled \
-DWITH_SSL=bundled \
-DENABLED_LOCAL_INFILE=1 \
-DWITH_EMBEDDED_SERVER=1 \
-DENABLE_DOWNLOADS=1 \
-DWITH_DEBUG=0

make && make install
ln -s /usr/local/mysql-5.6.36/ /usr/local/mysql

设置“bind-address”值为控制节点的管理网络IP地址和字符集等

\cp support-files/my*.cnf /etc/my.cnf

创建并编辑/etc/my.cnf.d/openstack.cnf
[mysqld]
bind-address = 10.0.10.11   #设置监听IP地址或0.0.0.0

default-storage-engine = innodb
innodb_file_per_table = on
max_connections = 4096
collation-server = utf8_general_ci
character-set-server = utf8

初始化

#初始化mysql
/usr/local/mysql/scripts/mysql_install_db --basedir=/usr/local/mysql/ --datadir=/usr/local/mysql/data --user=mysql

chown -R mysql.mysql /usr/local/mysql/
cp support-files/mysql.server /etc/init.d/mysqld
chmod 700 /etc/init.d/mysqld
chkconfig mysqld on
chkconfig --list mysqld
mkdir -p /usr/local/mysql-5.6.36/data   #创建mysql socket存放目录
chown -R mysql:mysql /usr/local/mysql-5.6.36/data #修改目录权限
/etc/init.d/mysqld start
netstat -lntup |grep 3306

设置密码并清理无用数据库

配置环境变量
echo "export PATH=$PATH:/usr/local/mysql/bin" >>/etc/profile
source /etc/profile

mysqladmin -u root -p password '123456'
mysql -uroot -p123456

mysql> drop user ''@'localhost';
mysql> drop user ''@'localhost.localdomain';
mysql> drop user 'root'@'127.0.0.1';
mysql> drop user 'root'@'::1';
mysql> drop database test;

配置RabbitMQ(在OpenStack架构中是交通枢纽的一个作用)

这里使用外部RabbitMQ集群,参考:RabbitMQ集群部署

集群信息

RabbitMQ集群主节点：10.0.10.21:5672
RabbitMQ Web Management: http://10.0.10.21:15672
账户密码：openstack

注：Web页面底部有个HTTP API，zabbix可以通过这个API来监控RabbitMQ

 生产环境需要配置集群，否则后期将成为瓶颈

注: 如果RabbitMQ无法启动，则需要检查hosts域名解析

阶段总结

MySQL和RabbitMQ都属于基础服务，基础服务是务必做高可用的，如果RabbitMQ基础服务Down了，会直接影响到OpenStack创建新的虚拟机，但是已经在运行的虚拟机，是不受影响的。
