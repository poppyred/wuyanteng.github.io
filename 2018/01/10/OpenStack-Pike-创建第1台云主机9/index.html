OpenStack-Pike-创建第1台云主机9

2018-01-10

####创建Provider Network网络

使用提供者（外部）网络，提供者网络通过L2（桥/交换机）设备连接到物理网络。这个网络包括为云主机提供IP地址的DHCP服务器。

注：如果选择“提供者网络”则只需创建一个公有网络。但如果选择“自服务网络”，需要同时创建一个公有网络和一个私有网络！切记！

创建Provider Networks

确保配置了以下参数

vim /etc/neutron/plugins/ml2/ml2_conf.ini
[ml2_type_flat] 
flat_networks  =  provider

------------------------------------------------
vim /etc/neutron/plugins/ml2/linuxbridge_agent.ini

[linux_bridge] 
physical_interface_mappings  =  provider：eth0
注：生产环境服务器有多个网卡，所以“提供者（外部）网络可能会配置在eth1网卡”。

source /scripts/admin_openrc

创建公共网络

openstack network create  --share --external --provider-physical-network provider --provider-network-type flat provider

参数释义：
--share 参数选项允许所有项目使用虚拟网络
--external选项将虚拟网络定义为外部

在公共网络上创建一个子网(注意:需要与现有机器同一网段,并确保DHCP不冲突)

openstack subnet create --network provider \
  --allocation-pool start=192.168.56.50,end=192.168.56.100 \
  --dns-nameserver 8.8.8.8 --gateway 192.168.56.2 \
  --subnet-range 192.168.56.0/24 provider

参数：
--allocation-pool  指定一个地址池
--dns-nameserver   指定一个DNS
--gateway 指定网关
--subnet-range  指定网络地址

创建一个主机规格,名称为mini.nano

openstack flavor create --id 0 --vcpus 2 --ram 768 --disk 5 mini.nano

注：使用Demo进行以下操作，这样生成的虚拟机都在Demo项目下

  • 生成一个key来实现无密码登陆虚拟云主机(可选)
source /scripts/demo-openrc

生成一个key或使用现有的key
ssh-keygen -q -N ""

添加key的公钥到计算服务中,并定义名称为mykey
openstack keypair create --public-key ~/.ssh/id_rsa.pub mykey

验证公钥

openstack keypair list
+-------+-------------------------------------------------+
| Name  | Fingerprint                                     |
+-------+-------------------------------------------------+
| mykey | 85:0f:61:f5:f8:ec:7b:86:7e:7a:c0:10:78:f8:01:29 |
+-------+-------------------------------------------------+
  • 增加安全组规则
开启icmp ping
openstack security group rule create --proto icmp default

允许ssh访问
openstack security group rule create --proto tcp --dst-port 22 default

启动一个虚拟机

  • 列出可用云主机类型
openstack flavor list
  • 列出可用镜像
openstack image list
+--------------------------------------+--------+--------+
| ID                                   | Name   | Status |
+--------------------------------------+--------+--------+
| 3e41628d-4dc9-4f73-8517-50dbc60ba5b4 | cirros | active |
+--------------------------------------+--------+--------+
  • 列出可用网络
openstack network list
+--------------------------------------+----------+--------------------------------------+
| ID                                   | Name     | Subnets                              |
+--------------------------------------+----------+--------------------------------------+
| 605a77b4-d232-4d0b-b915-d6572542c3d5 | provider | 925bb8f3-0e13-40d3-aa12-3c5e25d81238 |
+--------------------------------------+----------+--------------------------------------+

创建虚拟机

openstack server create --flavor mini.nano --image cirros \
 --nic net-id=605a77b4-d232-4d0b-b915-d6572542c3d5 \
 --security-group default \
 --key-name mykey First1Host

指定网络ID：--nic net-id=PROVIDER_NET_ID （使用provider公有网络的ID替换PROVIDER_NET_ID）
指定虚拟机的云主机规格mini.nano、image镜像:cirros、网络ID号、安全组和导入的免密公钥名称，最后命名这台云主机名称为First1Host

查看云主机状态
[root@controller ~]# openstack server list
+--------------------------------------+------------+--------+----------------------+--------+-----------+
| ID                                   | Name       | Status | Networks             | Image  | Flavor    |
+--------------------------------------+------------+--------+----------------------+--------+-----------+
| 6d3d639b-18a1-4bc2-8a71-6f4b4a95b444 | First1Host | ACTIVE | provider=172.16.1.13 | cirros | mini.nano |
+--------------------------------------+------------+--------+----------------------+--------+-----------+

当构建过程完全成功后，状态会从 BUILD``变为``ACTIVE

此时计算节点ifconfig命令结果如下：

[root@compute ~]# ifconfig
brq605a77b4-d2: flags=4163<UP,BROADCAST,RUNNING,MULTICAST>  mtu 1500
        inet 192.168.56.12  netmask 255.255.255.0  broadcast 192.168.56.255

eth0: flags=4163<UP,BROADCAST,RUNNING,MULTICAST>  mtu 1500
        inet6 fe80::20c:29ff:feca:1571  prefixlen 64  scopeid 0x20<link>
        ether 00:0c:29:ca:15:71  txqueuelen 1000  (Ethernet)

出现此结果说明创建云主机正常。

使用虚拟控制台访问虚拟机

获取云主机First1Host的VNC虚拟控制台地址
openstack console url show First1Host

将结果在浏览器中打开即可，账户:cirros   密码：cubswin:)

在云主机中验证

验证能否ping通公共网络的网关

$ ping -c 4 192.168.56.2

验证能否上网

$ ping -c 4 qq.com

验证云主机远程访问

ping -c 4 192.168.56.53

SSH访问

ssh  cirros@192.168.56.53

故障总结

虚拟机运行故障取决于两大组件，即neutron和nova,出了问题优先查这两个日志

云主机软重启

openstack server list
openstack server reboot SERVER       #软重启
