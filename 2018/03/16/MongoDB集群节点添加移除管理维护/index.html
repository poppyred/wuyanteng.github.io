MongoDB集群节点-添加移除管理维护

2018-03-16 
此操作在mongo V3.0已验证

1.集群运行时添加节点

查看复制集节点配置
./mongo 192.168.1.207:27017

db.version()    查看版本
 3.2.11 

rs0:PRIMARY> rs.conf();  查看集群配置文件
或
rs0:PRIMARY> rs.isMaster();

添加节点

添加新的数据目录
mkdir –p /mongodb/data/shard4

启动节点
./mongod –shardsvr –replSet shard1 –port 27019 –dbpath  /mongodb/data/shard4 –oplogSize 100 –logpath /mongodb/data/shard4.log –logappend –fork


连接主节点
mongo 192.168.1.207:27017

查看集群配置文件
rs0:PRIMARY> rs.isMaster();  

添加节点
 rs0:PRIMARY> rs.add("192.168.1.207:27019");
{ "ok" : 1 }

再次查看状态

再次查看状态
rs0:PRIMARY> rs.conf();
或
rs0:PRIMARY> rs.isMaster();

能看到新增节点说明添加成功

2. 设定节点优先级

优先级值越高,就是主节点

在新增节点的时候设定该节点的优先级别
repSetTest:PRIMARY> rs.add({"_id":3,"host":"localhost:27000","priority":1.5})

通过下面的方式修改优先级别

repSetTest:PRIMARY> var config=rs.config()

repSetTest:PRIMARY> config.members[2].priority=2
2
repSetTest:PRIMARY> rs.reconfig(config)
{ "ok" : 1 }

注意：第2步members大括号里面的取值(从0开始往下数,顺序值)和_id是没有关系的。
这些操作必须在Primary上进行。

3. Mongodb运行时-移除节点

连接主节点
mongo 192.168.1.207:27017

rs0:PRIMARY> rs.remove("192.168.1.207:27019");    此处输入移除命令

最后还需要关掉192.168.1.207:27019 该服务
Ps –ef | grep mongo 查找该服务
然后通过kill -9 pid 关闭服务, 测试没问题之后清理配置文件

4. MongoDB运行时移除分片

Mongodb运行时移除分片
1、连接mongos节点 ./mongo 192.168.1.207:30000/admin

2、运行db.runCommand( { removeshard: "your_shard_name" } )
{ msg : "draining started successfully", state: "started", shard :"mongodb0", ok : 1 }

3、查看状态，我们可以反复执行上面语句查看执行状态
db.runCommand( { removeshard: "your_shard_name" } )
{ msg: "draining ongoing", state: "ongoing", remaining: { chunks: 42, dbs : 1 }, ok: 1
说明正在迁移中。


4、移除非shard数据
db.runCommand( { movePrimary: "myapp", to: "mongodb1" })

这次就不是立即返回了，需要很久，然后会返回如下：
{ "primary" : "mongodb1", "ok" : 1 }


5、上面步骤都完成后，还需要再执行一次RemoveShard，清理残余数据。
db.runCommand( { removeshard: "mongodb0" } )
显示completed后，就可以安心的关闭mongod的进程了。
