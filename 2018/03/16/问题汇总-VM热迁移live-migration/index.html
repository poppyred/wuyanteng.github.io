问题汇总: VM热迁移live migration

2018-03-16

适用场景

热迁移: 在保证源和目标计算节点都存活的情况下,可进行热迁移。
但当计算节点宕机后,vm虚拟机并不能进行自动热迁移, 也可通过脚本检测VM存活然后去触发迁移

热迁移[Live Migration]前提条件

1. 源和目标节点的CPU类型要一致。 
2. 源和目标节点的Libvirt版本要一致。
3. 源和目标节点能相互识别对方的主机名称【即在/etc/hosts 中加入对方的条目】
4. 源和目标节点的 Libvirt TCP 远程监听服务得打开
   /etc/libvirt/libvirtd.conf
5. 在源和目标节点的nova.conf中指明在线迁移时使用 TCP 协议。
   /etc/nova/nova.conf

修改libvirt配置文件【所有节点】

/etc/libvirt/libvirtd.conf 修改为如下内容:

listen_tls = 0  
listen_tcp = 1  
auth_tcp = "none" 
listen_addr = "0.0.0.0" 
tcp_port = "16509"

/etc/sysconfig/libvirtd   修改为如下内容:

LIBVIRTD_ARGS="--listen"

/etc/libvirt/qemu.conf    修改为如下内容:

vnc_listen = "0.0.0.0"
user = "root"
group = "root"
dynamic_ownership = 1

修改nova配置【所有节点】

/etc/nova/nova.conf     修改为如下内容：

[libvirt]
live_migration_downtime = 500
live_migration_downtime_steps = 10
live_migration_downtime_delay = 75
live_migration_flag=VIR_MIGRATE_UNDEFINE_SOURCE,VIR_MIGRATE_PEER2PEER,VIR_MIGRATE_LIVE,VIR_MIGRATE_TUNNELLED

重启服务【所有节点】

#重启服务
systemctl restart libvirtd
systemctl restart openstack-nova-compute

热迁移-步骤【控制节点】

#查看VM实例
nova list

#查看当前VM运行在哪个节点上
nova show f3d749ba-98e1-4624-9782-6da729ad164c

#查看可用计算节点
nova service-list

#查看要迁移的节点的资源使用情况(CPU/内存/硬盘)
nova host-describe {Host}

#开始迁移(此命令不返回错误即为成功)
nova live-migration 要热迁移的VM_ID {Host}

注：如果不指明要迁移到的计算节点, 则使用nova自行调度
