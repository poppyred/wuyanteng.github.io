通过Logstash收集Tomcat日志

2017-12-13

基础环境配置

  • 配置Logstash
#安装
rpm -ivh logstash-5.6.5.rpm 

#配置软链
ln -s /etc/logstash /usr/share/logstash/config

#配置
vim /etc/logstash/logstash.yml 
path.config: /usr/share/logstash/config/conf.d     #修改path.config到这个目录


#启动
systemctl start logstash
systemctl status logstash
systemctl enable logstash

#软链启动命令
ln -s /usr/share/logstash/bin/logstash  /sbin

部署Gitlab+maven+jenkins持续集成环境-进阶1

2017-12-05

为什么要持续集成？

持续集成，简称CI。持续集成指的是，频繁的（一天多次）将代码集成到主干

好处：极大程度降低 “代码合并冲突” 几率

开发上传自己的代码到Gitlab, Gitlab发消息给Jenkins，随后Jenkins从仓库拉取代码，最后全自动部署到测试服务器进行相关测试，并将测试结果通知运维和开发，这种自动测试的方法叫做持续交付。

持续部署是持续交付的下一步，指的是代码通过评审以后，自动部署到生产环境

行话：CI/CD 指的是持续集成/持续交付/持续部署
