脚本编写-考虑点

(1) 有没有脚本放在同一目录
(2) 权限: 用哪个用户执行文件(加判断)
(3) 清空错文件怎么办（加判断）
(4) 错误提示：有没有执行成功知不知道（脚本执行完文字输出）
(5) 脚本通用性(即换个系统能否执行？环境判断)

sh与bash有区别吗？

通过命令:ls -l `which sh`  可以看到sh命令是bash的软链,所以没有区别

检查bash是否有漏洞

env x='() { :;}; echo be careful' bash -c "echo this is a test"

若出现两行结果就说明bash存在漏洞。

编辑shell文件自动添加shell注释等信息

[root@zyops ~]# cat .vimrc 

autocmd BufNewFile *.py,*.cc,*.sh,*.java exec ":call SetTitle()"

func SetTitle()  
    if expand("%:e") == 'sh'  
        call setline(1,"#!/bin/bash")
        call setline(2, "##############################################################")  
        call setline(3, "# File Name: ".expand("%"))
        call setline(4, "# Version: V1.0")
        call setline(5, "# Author: Gary Wu")
        call setline(6, "# Organization: https://wuyanteng.github.io")
        call setline(7, "# Created Time : ".strftime("%F %T"))
        call setline(8, "# Description:")
        call setline(9, "##############################################################")
        call setline(10, "")
    endif  
endfunc

shell脚本执行

#执行shell
方式一：sh  test.sh(推荐使用) 或 bash test.sh
方式二：source test.sh 或 . test.sh

区别：
sh 脚本名称   #是创建一个新的进程(窗口)执行文件中的命令
source 脚本名称 #直接在当前进程(窗口)执行文件中的命令

共同点：都能执行脚本命令

变量

全部变量（又名环境变量）
局部变量（普通变量）

查看系统全局变量
env

定义全局变量的方法：

export A=1
echo $A
env|grep A     #查看环境变量

#以上操作只在当前shell生效，如果需要长期生效需要追加到/etc/profile中
echo "export A=1" >>/etc/profile
source /etc/profile

#临时删除环境变量
unset A

#永久删除环境变量
在/etc/profile中删除并source即可

为什么要定义变量？

多次被重复使用的命令就可以使用变量替代

shell特殊变量

位置变量

echo $?   #检查上一个命令执行结果，0表示成功，1表示失败
echo $0   # 显示当前执行脚本名称(显示sh命令后接的文件名)；
echo $1 $2 .... $n (n 为无穷大的值)  #表示脚本后面接的参数，按位置数过去
echo $#        #输出命令行传参的个数,用法如: [$# -eq 2]
echo "$@"      #获取命令行全部传入参数[推荐使用]

echo $@        # 获取命令行全部传入参数[不推荐使用]
echo $*        # 获取命令行全部传入参数[不推荐使用]

echo $0
echo "第1个参数：" $1
echo "第2个参数：" $2
echo "第3个参数：" $3
echo "第4个参数：" $4
echo "第5个参数：" ${11}
echo "传入参数总数量：" $#
echo "获取全部传入参数：" “$@”
