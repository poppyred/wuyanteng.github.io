zabbix监控Windows

zabbix监控Windows

#导出windows性能计数器key，cmd命令行输入如下指令：
typeperf -qx > c:\windows-item.txt

#进入指定目录安装、启动
C:\zabbix_agents_3.0.0.win\bin\win64>zabbix_agentd.exe --install
C:\zabbix_agents_3.0.0.win\bin\win64>zabbix_agentd.exe --start
C:\zabbix_agents_3.0.0.win\bin\win64>zabbix_agentd.exe --stop

#配置zabbix_agent.conf指定zabbix server地址
Server=192.168.56.11,192.168.11.211

#添加windows主机

#测试取值
zabbix_get -s 192.168.56.11 -k perf_counter["\Process(Idle)\% Processor Time"]

#添加自定义监控项：
方式1：把自定义key写入到配置文件内（不推荐）
方式2：windows 监控项内添加zabbix web界面的windows模板key中:
key: perf_counter["\Process(Idle)\% Processor Time"]

#zabbix Web界面查看刚刚添加的最新数据

zabbix agent主动与被动-数据收集

默认即被动数据收集，即Server轮询向agent索要数据。

主动模式数据收集,即客户端agent定时向Server上报数据，新增配置如下：
ServerActive=192.168.56.11
Hostname="实际agent主机名"

注：zabbix agent的主动与被动模式可以同时并存

模板克隆+批量更新agent监控类型

(1)客户端改为agent主动数据收集模式后，zabbix无主动模式的模板，这时候可以完全克隆一个模板，如：Temple OS Linux的模板克隆后名称定义为：Temple OS Linux Active.
(2)然后批量修改克隆后的模板监控项类型为 zabbix agent(active) 
(3)最后修改主机使用的模板为Temple OS Linux Active即可

zabbix分布式监控之zabbix-proxy

zabbix-proxy介绍

Zabbix Proxy 可以代替 Zabbix Server 检索客户端的数据，然后把数据汇报给 Zabbix Server，
并且在一定程度上分担了 Zabbix Server 的压力。Zabbix Proxy 可以非常简便的实现了集中式、分布式监控。

Zabbix Proxy 是一个数据收集器，它不计算触发器、不处理事件、不发送报警。其端口号与server一样均为：10051

zabbix-proxy使用场景

监控远程区域设备
监控本地网络不稳定区域
当 Zabbix 监控上千设备时，使用它来减轻 Server 的压力
简化 Zabbix 的维护

注意事项：

(1)Zabbix Proxy需要数据库支持，并且必须和 Zabbix Server 分开，否则数据会被破坏，因为这两个数据库的表大部分都相同。

(2)Zabbix Proxy 收集到数据之后，首先将数据缓存在本地，然后在一定的时间之后传递给 Zabbix Server。
这个时间由 Zabbix Proxy 配置文件中参数 ProxyLocalBuffer and ProxyOfflineBuffer 决定。

(3)客户端使用agent active模式时，一定要记住在agent的配置文件参数ServerActive加上proxy的IP地址。切记！

(4)提前规划，尽量确保zabbix-server/zabbix-agent/zabbix-proxy/zabbix-get等版本一致，否则会出现无法预知的错误

源码安装zabbix-proxy

#下载
wget http://pkgs.fedoraproject.org/repo/pkgs/zabbix/zabbix-3.0.0.tar.gz/fd4032444711ebb45e92b4cd54a354c6/zabbix-3.0.0.tar.gz

#安装 zabbix 编译编译环境
yum -y install mysql-devel libxml2-devel net-snmp-devel libcurl-devel php-mysql libXpm php-bcmath php-gd php-mbstring php-xml t1lib gcc open-ssl

#源码编译
yum install -y curl-devel net-snmp-devel
groupadd zabbix && useradd -g zabbix zabbix
tar -zxvf zabbix-3.0.0.tar.gz && cd zabbix-3.0.0
./configure --prefix=/usr/local/zabbix --enable-agent --enable-proxy  --enable-get  --with-mysql --with-net-snmp --with-libcurl --with-libxml2
make && make install

#配置环境变量
cd /usr/bin && ln -s /usr/local/zabbix/bin/* ./
cd /usr/sbin && ln -s /usr/local/zabbix/sbin/* ./

#创建zabbix用户
useradd zabbix -s /sbin/nologin -M

安装MariaDB

yum install -y mariadb*

#启动MariaDB
systemctl restart mariadb
systemctl status mariadb

#初始化MariaDB
mysql_secure_installation   #初始密码空

#创建数据库并授权
create database zabbix_proxy;
grant all on zabbix_proxy.* to 'root'@'192.168.56.%' identified by '12345';
flush privileges;
quit;

导入zabbix默认数据库文件

mysql -uroot -p12345 zabbix_proxy < zabbix-3.0.0/database/mysql/schema.sql  #导入proxy数据文件

修改zabbix_proxy配置文件

Server=192.168.56.11              #zabbix服务端IP，注意是ServerIP
Hostname=zabbix_proxy　　          #必须和WEB页面添加代理时设置的名称一致
LogFile=/var/log/zabbix_proxy.log  #日志文件路径
DBHost=192.168.56.13　　           #zabbix_proxy数据库IP
DBName=zabbix_proxy                #zabbix_proxy数据库名
DBUser=root　　                    #zabbix_proxy数据库用户名
DBPassword=12345　　               #zabbix_proxy数据库密码
ConfigFrequency=60                 #配置文件同步间隔
DataSenderFrequency=5              #数据同步间隔

touch /var/log/zabbix_proxy.log
chown -R zabbix.zabbix /var/log/zabbix_proxy.log

启动zabbix_Proxy服务

zabbix_proxy -c /usr/local/zabbix/etc/zabbix_proxy.conf

此时，便可以在zabbix管理站点上配置proxy了，方法如下：

添加proxy节点

管理 – agent代理程序 – 创建代理

agent代理程序名称：  需要与zabbix_proxy节点配置文件中定义的Hostname一致
proxy mode:  选择proxy模式，zabbix proxy默认为主动模式
             填写zabbix_proxy节点的IP地址与端口号,zabbix_proxy端口号为10051

主机：  此处不建议批量添加，往下看            
描述：  描述清楚节点位置、用途等即可

重启agent服务

zabbix_get -s 192.168.56.12 -p 10050 -k "agent.ping"    #返回数字1说明OK
