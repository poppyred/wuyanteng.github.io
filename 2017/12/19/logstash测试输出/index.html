logstash测试输出

2017-12-19 
在配置ELK logstash的时候，为了降低错误，在收集的log输出到kafka或redis或elasticsearch之前，需要测试log是否正常

以nginx 测试输出为例

vim /etc/logstash/conf.d/nginx.conf

input {
  file {
    path => "/var/log/nginx/access.log"
    start_position => "end"
    type => "nginx-accesslog"
    codec => json
  }
}


output {
    stdout {
        codec => rubydebug      #输出模式: rubydebug
    }
}
  • 检测logstash配置文件语法并重启服务
logstash -f /usr/share/logstash/config/conf.d/nginx.conf -t
  • 前台启动logstash
#前台测试输出务必stop logstash服务
systemctl stop logstash   

#前台启动logstash
logstash -f /usr/share/logstash/config/conf.d/nginx.conf


刷新nginx测试页面,等待屏幕输出，如果输出的是json格式说明OK

OK, 接下来就配置输出到kafka或elasticsearch吧。若中间出了错误，就检查之前的步骤即可
