通过Logstash收集Tomcat日志

2017-12-13

基础环境配置

  • 配置Logstash
#安装
rpm -ivh logstash-5.6.5.rpm 

#配置软链
ln -s /etc/logstash /usr/share/logstash/config

#配置
vim /etc/logstash/logstash.yml 
path.config: /usr/share/logstash/config/conf.d     #修改path.config到这个目录


#启动
systemctl start logstash
systemctl status logstash
systemctl enable logstash

#软链启动命令
ln -s /usr/share/logstash/bin/logstash  /sbin

再Tomcat服务器上收集Tomcat日志

  • 配置java环境并部署Tomcat
#安装jdk
yum install -y java-1.8.0*
java -version


#下载部署Tomcat
cd /usr/local/src 
wget https://archive.apache.org/dist/tomcat/tomcat-8/v8.0.38/bin/apache-tomcat-8.0.38.tar.gz
tar xvf apache-tomcat-8.0.38.tar.gz
ln -sv /usr/local/src/apache-tomcat-8.0.38 /usr/local/tomcat

#配置启动
cd /usr/local/tomcat/webapps/
mkdir /usr/local/tomcat/webapps/webdir
echo "Hello,Tomcat" > /usr/local/tomcat/webapps/webdir/index.html
../bin/catalina.sh start && ss -lntup | grep 8080
  • 确认Tomcat Web正常访问

Tomcat日志格式转成json格式

vim /usr/local/tomcat/conf/server.xml
搜索“Valve”字段，删除并添加如下：

        <Valve className="org.apache.catalina.valves.AccessLogValve"
directory="logs"
        prefix="tomcat_access_log" suffix=".log"
        pattern="{&quot;clientip&quot;:&quot;%h&quot;,&quot;ClientUser&quot;:&quot;%l&quot;,&quot;authenticated&quot;:&quot;%u&quot;,&quot;AccessTime&quot;:&quot;%t&quot;,&quot;method&quot;:&quot;%r&quot;,&quot;status&quot;:&quot;&quot;,&quot;SendBytes&quot;:&quot;%b&quot;,&quot;Query?string&quot;:&quot;%q&quot;,&quot;partner&quot;:&quot;%{Referer}i&quot;,&quot;AgentVersion&quot;:&quot;%{User-Agent}i&quot;}"/>

./bin/catalina.sh stop   #停止Tomcat
cd /usr/local/tomcat/ && rm -rf logs/*   #清除原日志
./bin/catalina.sh start  #启动并访问Tomcat页面
http://10.0.10.24:8080/webdir/   #web访问
  • 查看新的日志文件
[root@localhost tomcat]# tailf logs/tomcat_access_log.2017-12-13.log   

{"clientip":"192.168.10.1","ClientUser":"-","authenticated":"-","AccessTime":"[13/Dec/2017:08:49:52 -0500]","method":"GET /webdir HTTP/1.1","status":"","SendBytes":"-","Query?string":"","partner":"-","AgentVersion":"Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/62.0.3202.94 Safari/537.36"}

可以看到，以上日志看起来很像json格式

如何获取json日志中的IP？

Python 脚本如下：

#!/usr/bin/env python
#coding:utf-8
#Author Zhang Jie
data={"clientip":"192.168.10.1","ClientUser":"-","authenticated":"-","AccessTime":"[13/Dec/2017:08:49:52 -0500]","method":"GET /webdir
HTTP/1.1","status":"","SendBytes":"-","Query?string":"","partner":"-","AgentVersion":"Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/62.0.3202.94 Safari/537.36"}
ip=data["clientip"]
print ip

vim /etc/logstash/conf.d/tomcat.conf

input {
  file {
    path => "/usr/local/tomcat/logs/tomcat_access_log.*.log"   #匹配所有日期log
    start_position => "end"                    #监听文件的起始位置，默认是end
    type => "tomcat-access-log"                #定义事件的唯一类型
    stat_interval => "3"
  }
  file {
    path => "/var/log/messages"
    start_position => "end"
    type => "messages-log"
    stat_interval => "3"
  }
}

output {
  if [type] == "tomcat-access-log" {       #判断type类型的value，并写入相同(或不同)的ES集群
    elasticsearch {
      hosts => ["10.0.10.21:9200"]     #定义写到主es机器,注意:需要把双引号写入到中括号的里面
      index => "logstash-tomcat-10.24-access-%{+YYYY.MM.dd}"   #指定收集到的日志索引名称
      codec => "json"            #告诉logstash需要处理tomcat的json格式日志
    }}
  if [type] == "messages-log" {
    elasticsearch {
      hosts => ["10.0.10.23:9200"]      #定义写到同集群其他es机器(试试效果而已)
      index => "messages-log-10.24-access-%{+YYYY.MM.dd}"
    }}

}
  • 测试配置文件
logstash -f /usr/share/logstash/config/conf.d/tomcat.conf -t
Configuration OK
  • 重启logstash服务并查看日志
systemctl restart logstash
tailf /var/log/logstash/logstash-plain.log

注：如果log日志提示某个文件访问拒绝(如：/var/log/messages: Permission denied), 需要修改文件权限
chmod 644  /var/log/messages
systemctl restart logstash   #再次重启logstash
  • 访问Tomcat Web页面让其生成日志
http://10.0.10.24:8080/webdir       #通过访问页面让其生成Tomcat日志,可以多刷新几次
echo "Test!" >> /var/log/messages   #通过追加内容模拟/var/log/messages生成日志

阶段总结

1. logstash的output中，输入同一个集群的不同机器上效果是一致的，因为他们是一个集群。当然也可以指定输出到不同的集群中。
2. 从kibana Tomcat页面中可以看出，Tomcat日志，已经变成了json格式，这是由于logstash中指定了参数codec="json" 的结果，否则kibana这里就是一坨
3. 这个坑是关于时间同步的，如果kibana与你的logstash时间不同步，kibana上的日志可能无法显示，这就需要养成做标准化的习惯，切记
