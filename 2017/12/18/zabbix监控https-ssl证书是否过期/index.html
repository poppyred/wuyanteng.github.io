zabbix监控https_ssl证书是否过期

2017-12-18

业务需求

https证书在到期前半个月进行高级提示，便于续费。

脚本

cat /server/scripts/cert_check.sh

#!/bin/sh
### SSL Certificate Expire Day Check Script ###
if [ "$1" = '' ];then
    echo "Need URL."
    exit 1;
fi
TARGET_URL=$1
EXP_DAY=`openssl s_client -connect ${TARGET_URL}:443 < /dev/null 2> /dev/null | openssl x509 -text 2> /dev/null | grep "Not After" | sed -e 's/^ *//g' | cut -d " " -f 4,5,6,7,8`
NOW_TIME=`date +%s`
EXP_TIME=`date +%s -d "${EXP_DAY}"`
if [ "${EXP_DAY}" != '' -a ${NOW_TIME} -lt ${EXP_TIME} ]; then
    echo $(((EXP_TIME-NOW_TIME)/(60*60*24)))
else
    echo "ERROR"
    exit 1;
fi

配置自定义key

假如在172.0.0.110机器来监控所有的SSL证书，在这台服务器配置自定义key
cd /etc/zabbix/zabbix_server.conf.d

vim cert_check.conf  #添加如下参数

UserParameter=cert_check[*],/bin/bash /server/scripts/cert_check.sh $1

zabbix Web配置

  • 创建模板
组态 -- 模板 -- 创建模板 -- 模板名称：SSL Cert  -- 可见名称：SSL证书监控 -- 群组选择所有 -- 主机/模板在..之中：选择“172.0.0.110”
  • 创建应用集
进入刚刚创建的模板中 -- 应用集 -- 创建应用集 -- 此处可根据企业域名创建多个应用集

如：*.exaple.cn
    *.exaple.com
  • 创建监控项
进入刚刚创建的模板中 -- 监控项 -- 创建监控项 -- 名称："cert_check $1"  -- 类型：zabbix代理 -- 键值: "cert_check[online.exaple.cn]" -- 应用集: 选择"*.exaple.cn"  -- 勾选"已启用" -- 创建/更新

其他域名监控项与此类似，区别是键值与应用集的不同
  • 创建触发器
进入刚刚创建的模板中 -- 触发器 -- 创建触发器 -- 名称：定义有意义即可 -- 
表达式:"{cert_check[online.exaple.cn].prev()}<15"  -- 创建/更新

触发器说明：online.exaple.cn域名证书有效期小于15天

其他域名监控项与此类似，区别是表达式中的域名不同
  • 在172.0.0.110监控中应用刚刚创建的模板
