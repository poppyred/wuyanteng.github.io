case条件语句

如果$x等于”值1”则执行命令1…以此类推

case $x in    
     值1)
         命令1
         ;;
     值2)
         命令2
         ;;

     ...

      *)
         命令n
         ;;

esac

使用场景：系统服务的启动脚本或菜单脚本一般使用case语句

#!/bin/bash

[ -f /usr/sbin/rsync ] || exit

case $1 in
     start)
          rsync --daemon
          ;; 

     stop)
          pkill rsync
          ;;

          sleep 2

          ;;

          echo "USAGE: sh $0 start|stop|reload"
          ;;          
esac

  • shell脚本限制DDos攻击
1.防止服务器被DDoS攻击的Shell脚本
　　mkdir /root/bin
　　vi /root/bin/dropip.sh
　　#!/bin/bash
　　/bin/netstat -na|grep ESTABLISHED|awk '{print $5}'|awk -F: '{print $1}'|sort|uniq -c|sort -rn|head -10|grep -v -E '192.168|127.0'|awk '{if ($2!=null && $1>4) {print $2}}'>/tmp/dropip
　　for i in $(cat /tmp/dropip)
　　do
　　/sbin/iptables -A INPUT -s $i -j DROP
　　echo “$i kill at `date`”>>/var/log/ddos
　　Done

脚本说明;在这段脚本中，最重要的是第二行：意思为获取ESTABLISHED连接数最多的前10个ip并写入临时文件/tmp/dropip,排除了内部ip段192.168|127.0开头的.通过for循环将dropip里面的ip通过iptables全部drop掉，然后写到日志文件/var/log/ddos

2.增加执行权限
　chmod +x /root/bin/dropip.sh

3.添加到计划任务，每分钟执行一次
　crontab -e
　*/1 * * * * /root/bin/dropip.sh

exit n

while true
   do 
   exit
done
作用：一旦条件满足,则脚本立即退出；其中exit中的n，意思是定义退出返回值

break n

作用：一旦条件满足，则跳出循环(注：if是判断不是循环)，但会继续执行循环之外的内容
其中n是指退出几层循环

continue n

作用：一旦条件满足，则结束当前本次循环,继续下一次循环
其中n是指退出几层循环

reture n

作用：只在函数中起作用,用来判断函数执行是否正确。和exit一样。

实现shell文件锁

不让脚本在执行的过程中重复执行的问题，可以使用shell文件锁解决。

#!/bin/bash

# 脚本初始化
function scripts_init(){
  prog=`basename $0 .sh`
  LockFile=/var/lock/subsys/${prog}.lock   # 使用锁文件
  LogFile=/var/log/${prog}.log   # 脚本记录日志
  PidFile=/var/run/${prog}.pid   # 记录进程号，可以管理脚本

  [ -f $LockFile ] && echo "There $LockFile is exist!!" && exit 1 ||touch $LockFile
  [ ! -f $LogFile ] && touch $LogFile
  [ -f $PidFile ] && echo "There $PidFile is exist!!" && exit 2|| echo $$ > $PidFile
}

# 记录日志
function writelog(){
  Date=$(date "+%F_%T")
  ShellName=`basename $0`
  Info=$1
  echo "$Date : ${ShellName} : ${Info}" >> ${LogFile}
}

# 脚本退出扫尾
function closeout(){
  [ -f $LockFile ] && rm -f $LockFile 
  [ -f $PidFile ]&& rm -f $PidFile
}
