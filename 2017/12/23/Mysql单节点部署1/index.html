数据库分类

DBMS 指数据库管理系统，分为RDBMS(关系型数据库管理系统)和NoSQL(非关系型数据库)

数据库世界排名统计网站：https://db-engines.com/en/ranking,根据排名趋势可以很清楚的知道研究方向

NoSQL产品代表:

键值存储：memcache、redis
列存储：HBASE
文档数据库: Mongodb(最接近关系型数据库)
图形存储：Neo4j 
通常以json格式进行数据存储

DBMS数据库产品代表

Oracle 、Mysql 、MariaDB、PerconaDB

MySQL启动过程

1. 启动后台守护进程，并生成工作线程
2. 提供预分配内存结构供MySQL处理数据使用

什么是实例？

MySQL的后台进程+线程+预分配内存结构

MySQLD服务器程序构成

连接层 --> SQL层 --> 存储引擎层(磁盘/内存/网格)

连接层：即客户端输入的SQL语句,如: select * from student
SQL层: 语法解析器解析SQL语句为 执行计划(即解析为SQL能理解的行为模式),授权并通过内部优化器自我判断(即自我判定选出一个成本最低的执行计划方式)，然后查询执行,执行后把结果交给存储引擎层（磁盘/内存/网络）进行数据调用，最后把结果返回给应用程序

MySQL启动流程

部署Mysql-V5.6

编译安装


#安装依赖包
yum groupinstall "Development Tools"
yum install ncurses-devel libaio-devel -y
rpm -qa ncurses-devel libaio-devel

#安装预编译工具cmake
yum install -y cmake

#创建用户
useradd -s /sbin/nologin -M mysql
id mysql

#官方下载mysql-版本:mysql-5.6.36
[root@MBJ-11 ~]# md5sum mysql-5.6.36.tar.gz   #检查md5是否与官方一致
a36a241164c78286993cbe1079626cdd  mysql-5.6.36.tar.gz

#解压、进入目录编译
tar -zxvf mysql-5.6.36.tar.gz && cd mysql-5.6.36

#注: cmake的作用：
1.定制软件的安装路径
2.定制mysql的源程序和命令脚本
    bin/mysqld (二进制的) 关键守护程序源码
    bin/mysql
    bin/mysqld_safe
    scripts/mysql.server
    bin/mysqldump
    bin/mysqladmin
    supports-file/mysql_install_db

mysql编译命令
cmake . -DCMAKE_INSTALL_PREFIX=/usr/local/mysql-5.6.36 \
-DMYSQL_DATADIR=/usr/local/mysql-5.6.36/data \
-DMYSQL_UNIX_ADDR=/usr/local/mysql-5.6.36/data/mysql.sock \
-DDEFAULT_CHARSET=utf8 \
-DDEFAULT_COLLATION=utf8_general_ci \
-DWITH_EXTRA_CHARSETS=all \
-DWITH_INNOBASE_STORAGE_ENGINE=1 \
-DWITH_FEDERATED_STORAGE_ENGINE=1 \
-DWITH_BLACKHOLE_STORAGE_ENGINE=1 \
-DWITHOUT_EXAMPLE_STORAGE_ENGINE=1 \
-DWITH_ZLIB=bundled \
-DWITH_SSL=bundled \
-DENABLED_LOCAL_INFILE=1 \
-DWITH_EMBEDDED_SERVER=1 \
-DENABLE_DOWNLOADS=1 \
-DWITH_DEBUG=0

如果上面编译出错，重新解压mysql源码包，进入目录重新cmake编译
make && make install
ln -s /usr/local/mysql-5.6.36/ /usr/local/mysql

初始化及创建数据库

\cp support-files/my*.cnf /etc/my.cnf

#初始化mysql
/usr/local/mysql/scripts/mysql_install_db --basedir=/usr/local/mysql/ --datadir=/usr/local/mysql/data --user=mysql

chown -R mysql.mysql //usr/local/mysql/
cp support-files/mysql.server /etc/init.d/mysqld
chmod 700 /etc/init.d/mysqld
chkconfig mysqld on
chkconfig --list mysqld
mkdir -p /usr/local/mysql-5.6.36/data   #创建mysql socket存放目录
chown -R mysql:mysql /usr/local/mysql-5.6.36/data #修改目录权限
/etc/init.d/mysqld start
netstat -lntup |grep 3306

配置环境变量

echo  'export PATH=$PATH:/usr/local/mysql/bin' >>/etc/profile
tail -1 /etc/profile
source /etc/profile

常见问题

常见问题:
故障:ERROR! The server quit without updating PID file 
解决方法：创建存放pid的目录并使用“chown -R mysql.mysql”命令,修改pid存放目录的权限

kill/pkill关停数据库或mysql服务器意外终止，造成启动失败

配置文件读取顺序

mysql配置文件读取顺序

/etc/my.cnf -->  /etc/mysql/my.cnf -->  $MYSQL_HOME/my.cnf --> ~/.my.cnf  
--> --defaults-extra-file  -->  --defaults-file  最后读取命令行上其他的配置

单独定义一个文件来作为mysql socket

/etc/init.d/mysqld stop   #停止服务

vim /tmp/aaa.txt   #创建并配置文件
[mysqld]
socket=/tmp/aaa.sock

/usr/local/mysql/bin/mysqld_safe --default-file=/tmp/aaa.txt   #启动

如果想回归源socket文件，只需要正常启动mysqld服务即可。

注：启动命令一旦使用了--default-file参数，其他mysql配置文件(即上面的配置文件读取顺序)均不会再读取
