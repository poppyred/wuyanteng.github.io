sersync+rsync实时同步

2017-08-26

sersync项目介绍

Github发布: https://github.com/wsgzao/sersync

原理

下载

解压

unzip sersync_installdir_64bit.zip
mv sersync_installdir_64bit /usr/local/sersync
cd /usr/local/sersync
mv sersync/* ./
rm sersync -rf

目录结构

bin目录保存的是sersync可用的命令
conf目录保存的是配置文件
logs目录保存的是log文件，默认空

编辑sersync配置文件

cd bin/ && chmod +x bin/sersync  #赋予命令可执行权限
cd conf/ && cp confxml.xml confxml.xml.bak

vim vim confxml.xml


#开启debug信息。
<debug start="true"/>


#inotify
<inotify>
    <delete start="true"/>
    <createFolder start="true"/>
    <createFile start="false"/>
    <moveTo start="true"/>
    <attrib start="false"/>
    <modify start="false"/>
</inotify>


#rsync同步信息配置
<rsync>
   <commonParams params="-az"/>
   <auth start="true" users="rsync_backup" passwordfile="/etc/rsync.password"/>
   <userDefinedPort start="false" port="874"/><!-- port=874 -->
   <timeout start="false" time="100"/><!-- timeout=100 -->
   <ssh start="false"/>
</rsync>

#配置rsync服务器主机IP和模块名称（如果本地监控/同步的目录如果有多个，需要新增以下字段）
<sersync>
    <localpath watch="/backup">  #配置本地监控同步的目录
        <remote ip="192.168.56.41" name="backup"/>  #rsync服务器IP和模块名称
        <!--<remote ip="192.168.8.39" name="tongbu"/>-->
        <!--<remote ip="192.168.8.40" name="tongbu"/>-->
     </localpath>
     <rsync>
         <commonParams params="-az"/>
         <auth start="true" users="rsync_backup" passwordfile="/etc/rsync.password"/>
         <userDefinedPort start="false" port="874"/><!-- port=874 -->
         <timeout start="false" time="100"/><!-- timeout=100 -->
         <ssh start="false"/>
      </rsync>


#重传机制，每隔多久进行重传，单位:分钟
<failLog path="/tmp/rsync_fail_log.sh" timeToExecute="60"/><!--default every 60mins execute once-->

启动sersync服务

./bin/sersync -h   #查看帮助

####################################################
参数-d:启用守护进程模式
参数-r:在监控前，将监控目录与远程主机用rsync命令推送一遍
c参数-n: 指定开启守护线程的数量，默认为10个
参数-o:指定配置文件，默认使用confxml.xml文件
参数-m:单独启用其他模块，使用 -m refreshCDN 开启刷新CDN模块
参数-m:单独启用其他模块，使用 -m socket 开启socket模块
参数-m:单独启用其他模块，使用 -m http 开启http模块
不加-m参数，则默认执行同步程序
##################################################### 

./bin/sersync -dro /usr/local/sersync/conf/confxml.xml      启动

查看进程

[root@nfs01 backup]# ps -ef |grep sersync

root       5771      1  0 11:07 ?        00:00:00 ./bin/sersync -dro /usr/local/sersync/conf/confxml.xml

bug

sersync+rsync同步的时候，NFS服务端rsync不能使用xinetd来启动，否则会触发bug，导致文件同步异常。

NFS服务端rsync需要使用: rsync --daemon的方式启动。
