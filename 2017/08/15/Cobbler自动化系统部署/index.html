Cobbler自动化系统部署

2017-08-15

Cobbler是一个Linux服务器安装的服务，可以通过网络启动(PXE)的方式来快速安装、重装物理服务器和虚拟机，同时还可以管理DHCP，DNS等。Cobbler可以使用命令行方式管理，也提供了基于Web的界面管理工具(cobbler-web)，还提供了API接口，可以方便二次开发使用。

cobbler工作原理

环境准备

1. 查看系统版本
   [root@cobbler-9 ~]# cat /etc/redhat-release 
   CentOS Linux release 7.3.1611 (Core) 

2. 内核版本
   [root@cobbler-9 ~]# uname -a
   Linux cobbler-9 3.10.0-514.el7.x86_64

3. 关闭selinux
   [root@cobbler-9 ~]# getenforce 
   Permissive
   [root@cobbler-9 ~]# cat /etc/selinux/config 
   SELINUX=disabled

4. 停止firewalld防火墙
   [root@cobbler-9 ~]# systemctl stop firewalld.service
   [root@cobbler-9 ~]# systemctl disable firewalld.service


5. 主机名
   [root@cobbler-9 ~]# hostname
   cobbler-9

6. 安装yum源
   mv /etc/yum.repos.d/CentOS-Base.repo /etc/yum.repos.d/CentOS-Base.repo.backup
   wget -O /etc/yum.repos.d/CentOS-Base.repo http://mirrors.aliyun.com/repo/Centos-7.repo
   yum clean all && yum makecache

安装cobbler

[root@cobbler-9 ~]# yum install cobbler cobbler-web pykickstart httpd dhcp tftp xinetd -y

cobbler          #Cobbler程序包
cobbler-web      #Cobbler的web服务包
pykickstart      #Cobbler检查kickstart语法错误
httpd            #Apache web服务
dhcp             #Dhcp服务
tftp             #Tftp服务

启动服务

cobbler的运行依赖于dhcp、tftp、rsync及dns服务。

[root@cobbler-9 ~]# systemctl start httpd
[root@cobbler-9 ~]# systemctl enable httpd

[root@cobbler-9 ~]# systemctl start cobblerd
[root@cobbler-9 ~]# systemctl enable cobblerd

[root@cobbler-9 ~]# ss -lntup |grep cobblerd

检查cobbler的配置

[root@cobbler-9 ~]# cobbler check

针对出现的问题逐一解决：

问题1：修改/etc/cobbler/settings文件中--提供cobbler服务的主机相应的IP地址或主机名
[root@cobbler-9 ~]# sed -i 's/server: 127.0.0.1/server: 10.10.0.10/' /etc/cobbler/settings
[root@cobbler-9 ~]# grep 10.10.0.10 /etc/cobbler/settings 
next_server: 10.10.0.10
server: 10.10.0.10

问题2：修改/etc/xinetd.d/tftp
修改disable = no

问题3：按照提示执行“cobbler get-loaders” 下载loaders
[root@cobbler-9 ~]# cobbler get-loaders
[root@cobbler-9 ~]# ls /var/lib/cobbler/loaders/  查看下载内容

问题4：按照提示执行"systemctl enable rsyncd"
[root@cobbler-9 ~]# systemctl start rsyncd
[root@cobbler-9 ~]# systemctl enable rsyncd

问题5：创建cobbler默认系统用户及密码
[root@cobbler-9 ~]# openssl passwd -1 -salt 'root' 'passwd'
$1$root$CZ.bdnz2SUozLTg0W7Yhm1
修改密码为刚创建的新密码
[root@cobbler-9 ~]# grep default_password /etc/cobbler/settings
default_password_crypted: "$1$root$CZ.bdnz2SUozLTg0W7Yhm1"

重启cobbler后再检查
[root@cobbler-9 ~]# systemctl restart cobblerd
[root@cobbler-9 ~]# cobbler check

此时，还剩下最后两项，一个是与debian系统相关，另一个是电影管理设备相关，此处暂不做调整。

配置DHCP

此参数为1时表示使用，cobbler管理dhcp
[root@cobbler-9 ~]# sed -i 's#manage_dhcp: 0#manage_dhcp: 1#g' /etc/cobbler/settings
[root@cobbler-9 ~]# vim /etc/cobbler/dhcp.template
#仅列出修改过的部分
subnet 10.10.0.0 netmask 255.255.255.0 {
     option routers             10.10.0.1;
     option domain-name-servers 10.10.0.1;
     option subnet-mask         255.255.255.0;
     range dynamic-bootp        10.10.0.100 10.10.0.254;
     #next-server               $next_server;
     next-server                10.10.0.10;

*定制化安装

1. Cobbler支持设备的物理MAC地址来区分设备，针对不同设备安装操作系统
2. 查看服务器MAC地址，并牢记
3. Cobbler可以自定义配置网络接口，通过system来固定机器的IP、掩码、网关、DNS、主机名、等等实现基础环境标准化

[root@cobbler-9 kickstarts]# cobbler system add --name=linux-Centos7.3-online --mac=00:50:56:38:F3:C5 --profile=CentOS-7.3-x86_64 \
--ip-address=10.10.0.10 --subnet=255.255.255.0 --gateway=10.10.0.1 --interface=eth0 \
--static=1 --hostname=centos7.3 --name-servers="10.10.0.1" \
--kickstart=/var/lib/cobbler/kickstarts/CentOS-7.3-x86_64.ks
#--name 自定义，但不能重复

查看定义的列表

[root@linux-node1 ~]# cobbler system list
linux-Centos7.3-online

打开MAC地址为”00:50:56:38:F3:C5”的机器发现会自动安装预选的操作系统
