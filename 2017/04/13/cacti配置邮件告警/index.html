2017-04-13 
cacti邮件告警
cacti插件安装
所需插件：
Setting插件              用于邮件报警
Thold插件                用于设备异常预警、阀值报警
monitor插件              主机状态监控

下载插件：
http://docs.cacti.net/plugin:thold
http://docs.cacti.net/plugin:settings
http://docs.cacti.net/plugin:monitor

安装插件：
tar zxvf thold-v0.5.0.tgz -C /usr/local/nginx/html/cacti/plugins
tar zxvf settings-v0.71-1.tgz -C /usr/local/nginx/html/cacti/plugins/
tar zxvf monitor-v1.3-1.tgz -C /usr/local/nginx/html/cacti/plugins/

注册到Cacti页签
[root@localhost ~]# vi /usr/local/nginx/html/cacti/include/config.php
$url_path = "/";    #在这行下面添加如下两行
$plugins[] = 'settings';
$plugins[] = 'thold';
启动插件
登陆Web -- console -- Plugin Management -- 点击向下箭头↓ 来安装插件3个插件 --- 启动插件

之后，Web控制栏会多出一些功能选项
配置发送邮件信息
console -- Settings -- Mail/DNS 

Test Email：用于发送到这个邮箱做测试
Mail Services: SMTP
From Email Address: 填写发送邮箱地址  
From Name: 填写发送邮箱地址
SMTP Hostname: exmail.qq.com
SMTP Port: 25
SMTP Username: 填写发送邮箱地址
SMTP Password: 填写邮箱密码
DNS：114.114.114.114
将Bitys转换为Mbits,便于设置阈值
默认Cacti报警使用Bites来监控流量的，需要转换成Mbits，这样方便设置阀值。

console -- GraphManagement -- CDEFs -- 填写：Add—Byte to Mbits  添加成如下所示

#####告警展示在Monitor界面

console -- settings -- Misc -- 勾选：Show Icon Legend
添加监控对象与报警信息
console -- Thresholds -- add

Host: 选择对应的HOST
Graph: 选择要监控的接口
Data Source: 选择traffic_in或traffic_out
点击Create创建

接下来分别设置traffic_in和traffic_out, 配置如下：

勾选：Threshold Enabled
Re-Alert Cycle: Every 30 Minutes   若故障持续存在,30分钟后，重复发送告警。
Warning High / Low Settings：根据实际情况修改, 这个是高于某个值发警告
Alert High / Low Settings：   这个严重等级更高
          High Threshold： 450
          Low Threshold：0
          Breach Duration：5 Minutes


套用刚才自定义的带宽模板
Data Type： CDEF
Threshold CDEF: Byte to Mbits

Alert Emails: 填写收件人邮箱
Warning Emails: 填写收件人邮箱
查看报警状态及测试报警
Web控制栏  -- thold
