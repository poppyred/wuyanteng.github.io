dingo反代Goolge DNS

2017-04-03 
访问Google一直是程序员乃至IT人士基本生存技能，今天聊聊无污染DNS之Dingo

准备条件

1.境外服务器一台
2.安装go环境

dingo安装

cd /opt
git clone https://github.com/pforemski/dingo.git
cd dingo && vim ./build.sh   修改编译后的二进制文件路径
DEST="/opt/dingo-$VERSION"

go get github.com/pforemski/dingo
./build.sh linux-amd64

编译完成后会在/opt目录下出现。执行dingo --help会出现dingo的参数说明
/opt/dingo-linux-amd64/dingo-linux-amd64 --help
软链
ln -s /opt/dingo-linux-amd64/dingo-linux-amd64 /usr/sbin/

dingo使用

启动（默认使用Google DNS）
dingo-linux-amd64 -port=5353 -bind=x.x.x.x 

注：-bind可以指定为境外服务器公网IP
    -port指定本机开放端口

测试

测试之前,iptables允许UDP 5353端口访问
dig twitter.com @x.x.x.x -p5353

开机自启动

vim /etc/rc.local
dingo-linux-amd64 -port=5353 -bind=x.x.x.x

高级使用

dingo-linux-amd64 -port=5353 -bind=x.x.x.x
上面这条启动命令默认使用的是Google DNS, 当然也可以代理使用自定义DNS, 命令如下：

dingo-linux-amd64 -bind 207.246.67.28 -port=5353 -odns:server=1.0.0.1,9.9.9.9
