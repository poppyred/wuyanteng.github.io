Nginx负载均衡

2017-09-23

集群分类

1.负载均衡集群LB
   硬件实现负载：F5硬件设备、A10硬件设备
   软件实现负载：LVS（4层负载均衡）/ HAproxy(4层/7层负载均衡) 
2. 高可用集群HA
3. 网络集群

负载均衡作用

1. 对用户的访问请求进行调度管理
2. 对用户的访问请求进行压力分担

反向代理与数据转发的区别

nginx反向代理-部署

web服务器配置

在多台web服务器部署nginx服务并完成测试访问

upstream配置

upstream模块：ngx_http_upstream_module  类似于一个池，将nginx节点放入池中
proxy_pass模块：ngx_http_proxy_module   用来调用upstream定义的名称

在http区域内定义负载池
    upstream server_pools {
         server 10.0.0.7:80 weight=3 max_fails=3 fail_timeout;
        server 10.0.0.8:80 weight=1 max_fails=3 fail_timeout backup;
    }     

    server {          #配置负载虚拟主机bbs
         listen 80;
         server_name bbs.etiantian.org;


    location / {
             proxy_pass http://server_pools;
            }

upstream模块

upstream模块功能：
    ①定义后端可调剂节点信息
    ②实现权重值负载访问功能-如：weight=2
    ③定义后端访问的失败次数，失败后就访问下一台服务器-如：max_fails=3
    ③定义后端访问失败重试超时时间，超过超时时间会重新请求此服务器-如：fail_timeout=10
    ④当集群中的负载节点服务器都挂了的情况下，使用热备节点,即backup

注：使用weight和backup时，不能使用ip_hash算法。

upstream调度算法：
     ①定义轮询调度算法- rr(默认调度算法)
     ②定义权重调度算法- wrr 
     ③定义静态调度算法-ip_hash   对用户源公网IP进行hash计算
     ④定义最小的连接数-least_conn  
注：IP_hash会造成负载不均的问题，但可以使用redis缓存去记录session来解决问题 

示例：
1、轮询（默认） 
每个请求按时间顺序逐一分配到不同的后端服务器，如果后端服务器down掉，能自动剔除。 

2、weight 
指定轮询几率，weight和访问比率成正比，用于后端服务器性能不均的情况。 
例如： 
upstream bakend { 
server 192.168.0.14 weight=10; 
server 192.168.0.15 weight=10; 
} 

3、ip_hash 
每个请求按访问ip的hash结果分配，这样每个访客固定访问一个后端服务器，可以解决session的问题。 
例如： 
upstream bakend { 
ip_hash; 
server 192.168.0.14:88; 
server 192.168.0.15:80; 
} 

4、fair（第三方） 
按后端服务器的响应时间来分配请求，响应时间短的优先分配。 
upstream backend { 
server server1; 
server server2; 
fair; 
} 

5、url_hash（第三方） 
按访问url的hash结果来分配请求，使每个url定向到同一个后端服务器，后端服务器为缓存时比较有效。 
例：在upstream中加入hash语句，server语句中不能写入weight等其他的参数，hash_method是使用的hash算法 

upstream backend { 
server squid1:3128; 
server squid2:3128; 
hash $request_uri; 
hash_method crc32; 
}

proxy_pass模块

server {
        listen 80;
        server_name www.etiantian.org;
        location / {
            proxy_pass http://server_pools;
            proxy_set_header host $host;
            proxy_set_header X-Forwarded-For $remote_addr;
        }
    } 

其中,
proxy_set_header host $host;  把用户访问的host头部信息改为用户输入的host域名信息
因为在upstream负载模式下，HTTP应用层-抓包过程可以看到，nginx前端负载服务器到nginx后端机的数据包头部，host信息为upstream池的名称

proxy_set_header X-Forwarded-For $remote_addr；  #显示访问客户端的真实源IP地址
注：需要提前在nginx.conf日志格式(log_format)区域中定义

nginx负载机器上进行如下配置：

worker_processes  1;
events {
    worker_connections  1024;
}
http {
    include       mime.types;
    default_type  application/octet-stream;
    sendfile        on;
    keepalive_timeout  65;                                  
    upstream server_pools {
        server 10.0.0.7:80;
        server 10.0.0.8:80;
    } 
    server {
        listen 80;
        server_name bbs.etiantian.org;
        location / {
            proxy_pass http://server_pools;
            proxy_set_header host $host;
            proxy_set_header X-Forwarded-For $remote_addr;
        }
    }
    server {
        listen 80;
        server_name www.etiantian.org;
        location / {
            proxy_pass http://server_pools;
            proxy_set_header host $host;
            proxy_set_header X-Forwarded-For $remote_addr;
        }
    }
}

基于URI进行转发-实现动静资源访问分离

例如：
/upload   10.0.0.8:80
/static   10.0.0.7:80
/         10.0.0.9:80

nginx配置文件中location区域类似判断功能，如果什么什么则怎样。

负载均衡服务器配置

worker_processes  1;
events {
    worker_connections  1024;
}
http {
    include       mime.types;
    default_type  application/octet-stream;
    sendfile        on;
    keepalive_timeout  65;
    log_format  main  '$remote_addr - $remote_user [$time_local] "$request" '
                      '$status $body_bytes_sent "$http_referer" '
                      '"$http_user_agent" "$http_x_forwarded_for"';       
    upstream upload_pools {         #定义一个upload后端集群池
    server 10.0.0.8:80;
              }

    upstream static_pools {        #定义一个static后端集群池
           server 10.0.0.7:80;
     }

    upstream default_pools {       #定义一个默认后端集群池
           server 10.0.0.9:80;
     }

server {
        listen 80;
        server_name www.etiantian.org;

   location /static/ {                     #当用户访问static下的URI时
         proxy_pass http://static_pools;   #调用对应后端集群池static_pools
         proxy_set_header Host $host;
         proxy_set_header X-Forwarded-For $remote_addr;
    }

    location /upload/ {                       #当用户访问upload下的URI时
            proxy_pass http://upload_pools;   #调用对应后端集群池upload_pools
            proxy_set_header Host $host;
            proxy_set_header X-Forwarded-For $remote_addr;
    }

     location / {                             #当用户访问/下的URI时
            proxy_pass http://default_pools;  #调用对应后端集群池default_pools
            proxy_set_header Host $host;
            proxy_set_header X-Forwarded-For $remote_addr;
         }
         access_log  logs/access_www.log  main;
    }
}

注：如果location判断不匹配，则默认匹配显示默认的/所对应的后端集群池中服务器站点信息

手机与电脑访问页面显示不同界面-实现

根据用户不同的浏览器客户端进行转发

lb01负载均衡配置

worker_processes  1;
events {
    worker_connections  1024;
}
http {
    include       mime.types;
    default_type  application/octet-stream;
    sendfile        on;
    keepalive_timeout  65;
    log_format  main  '$remote_addr - $remote_user [$time_local] "$request" '
                      '$status $body_bytes_sent "$http_referer" '
                      '"$http_user_agent" "$http_x_forwarded_for"';

upstream upload_pools {
  server 10.0.0.8:80;
}

upstream static_pools {
  server 10.0.0.7:80;
}

upstream default_pools {
  server 10.0.0.9:80;
}

    server {
        listen 80;
        server_name www.etiantian.org;
        location / {
         proxy_set_header Host $host;
         if ($http_user_agent ~* "Android")   #判断是Android手机，则请求static_pools后端集群
          {
            proxy_pass http://static_pools;
          }
         if ($http_user_agent ~* "iPhone")   #判断是iPhone手机，则请求static_pools后端集群
          {
            proxy_pass http://upload_pools;
          }
           proxy_pass http://default_pools;  #判断如果都不匹配，则请求默认后端集群
               }
         access_log  logs/access_www.log  main;
    }
}

注：proxy_set_header Host $host;  不能放在if区域内
~*  表示不区分大小写
