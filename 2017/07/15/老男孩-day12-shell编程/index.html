    什么是shell？

    shell, 即命令解释器.由命令、变量和流程控制语句有机的结合

CentOS默认shell是/bin/bash

    echo "df -h"
echo "df -h" |bash
把字符交给bash去执行，然后返回结果。

    文件类型-判断

    示例：
[ -f /etc/hosts ] && cp /etc/hosts /tmp || echo "error: 文件不存在"
[ -f /etc/host ] && echo "表达式成立" || echo "表达式不成立"

[ -f /etc/host ]  -f前面和host后面语法要求有空格

语法：
-f 判断文件是否存在
   例如：[ -f /etc/host ] && echo "表达式成立" || echo "表达式不成立"

-d 判断目录
   例如: [ -d /etc/host ] && echo "表达式成立" || echo "表达式不成立"

-r 判断当前用户是否有读取权限
   例如：[ -r /etc/shadow ] && echo "有可读权限" || echo "无可读权限"

-w 判断当前用户是否有写入权限
-x 判断当前用户是否有可执行权限
-e 判断目录或文件是否存在
-z 字符串长度为0则为真，否则不成立.【注：空格也不行】
   例如: [ -z "" ] && echo "表达式成立" || echo "表达式不成立" 表达式成立
-n 字符串长度不为0则为真，否则不成立
   例如：[ -n " " ] && echo "表达式成立" || echo "表达式不成立"

    “字符串1” = “字符串2”  若字符串1等于字符串2则为真，可以使用“==”代替“=”
“字符串1” != “字符串2”   若字符串1不等于字符串2则为真，但不能使用“!=="代替"!=”

    Bash中的“[[]]”与“[]”的区别
    "[[]]" 是“[]”的提高版本，但“[[]]”仅在Bash、Zsh和Korn Shell中可用，而“[]”几乎可以在任一种shell下使用。不过，像我们老版本的Gentoo都支持bash，所以还是建议使用双中括号的，了解以下区别即可。

具体不同点-示例：
[[ a > b ]] || echo "false"
[ a \> b ] || echo "false"

[[ az < za ]] && echo "az < za"
[  az \< za ] && echo "az < za"

也可以理解为，单括号内特殊字符有特殊含义，需要使用转义符(\)进行转义才行。

    if语句

    --单分支条件语句: 一个条件一个结果，可以使用单分支if语句

--双分支条件语句:  一个条件两个结果，可以使用双分支if语句

--多分支条件语句： 多个条件多个结果，可以使用多分支if语句

    传参方式-if语句判断-实例1：

    #!/bin/bash

num1=$1
num2=$2
[[ $# -ne 2 ]] && echo "请输入2个数字" && exit
if [[ $num1 > $num2 ]]
   then
       echo "$num1>$num2"
elif [[ $num1 < $num2 ]]
   then
       echo "$num1<$num2"
else
       echo "$num1=$num2"
fi

执行：sh if.sh 0 1 

注1：如果command与then写在同一行，command命令与then之间需要使用英文 “;” 隔开
     if语句结构一定要以fi结尾
#########################################################################################
注2：上面使用2个中括号是因为可以避免部分代码歧义，使shell更优雅。我们可以参考下面这个示例来了解歧义
a=;b=3;
if [ a -gt $b ]; then echo "true";fi

-bash: [: -gt: unary operator expected  //报错
而改为if [[ a > 3 ]]; then echo "true";fi  //不再报错
究其原因，是因为如果变量a值为空，那么就成了 [ -gt 3 ] ，显然 [ 和 $b 不相等并且缺少了 [ 符号，所以报了这样的错误。当然不总是出错，如果变量a值不为空，程序就正常了，所以就是这点歧义。

    读入赋值方式-if语句判断-实例2：

    #!/bin/bash

read  -sp  "请输入密码："  pass     

if test "$pass" == "garywu" ;then
   echo -e "\n恭喜，认证成功"
exit 0
fi

exit 1

注：
(1)test命令用于做各种测试，并当测试成功或失败时设置它的退出状态码为0（表示真）或1（表示假）
(2)echo -e命令开启了对特殊字符的支持，本例中\n是换行符
(3)使用read读取输入的密码（使用-s选项意思是输入的内容不会打印在终端，实现隐藏密码的目的）
(4)使用test命令比对两个字符串，如果$pass的密码为garywu则验证通过，打印“恭喜，认证成功”，并返回退出状态码0；如果$pass的内容不是garywu，则跳出if结构，退出并返回退出状态码1

##############################################

    嵌套的if/else语句-实例

    定义整型变量命令: declare -i

例如，希望得到45+20 的整数运算结果：
$ M=45+20  
$ echo $M  
45+20 
执行M=45+20 之后，M 的值为字符串45+20 ， 而不是希望得到的65 。执行M=$(45+20)，或let M=45+20 才能得到65。如果事先声明变量是整型数，就不必使用$((…))或者let
$ declare -i K  
$ K=45+20  
$ echo $K  
65 

    [root@root scripts]# vim qiantao.sh
#!/bin/bash

declare -i count        #定义一个整型变量count
read -p "请键入一个数字：" count      #把输入的数字给变量count

if [[ $count > 100 ]];then       
   echo "$count > 100"


else
    if [[ $count < 100 ]];then
    echo "$count < 100"

    else
      echo "$count = 100"

    fi
fi
#######################################

    多条件、多级的if/else语句-实例

    #!/bin/bash


if [[ $# -eq 0 ]];then
   echo "$0: You must give/supply one integers."
   exit 1
fi

if [[ $1 -gt 10 ]];then
   echo "$1>10"

elif [[ $1 -lt 10 ]];then
   echo "$1<10"

elif [[ $1 -eq 10 ]];then
   echo "$1=10"

else
    echo "Opps! $1 is not number,give number."

fi

首先判断脚本运行后，如果没指定参数则退出并返回错误码1；如果指定的参数大于0，则打印 $1>0 ;如果指定的参数小于0，则打印$1<0 ;如果指定的参数等于0，则打印$1=0;如果条件都不满足，打印“Opps! $1 is not number,give number.”

    while语句（又称死循环）

    善于执行守护进程，以及我们希望循环不退出持续执行。
小于1分钟的任务都可以使用while死循环，比如下面的例子，5秒打印一次hello world!
[root@oldboy202 scripts]# vim dead.sh
while true     
  do
  echo "hello world!"
  sleep 1
done

其中, true表示条件永远成立
    技巧：sh -x 参数会把脚本的执行过程也会显示出来，如：
    [root@oldboy202 scripts]# sh -x start.sh 
++ chkconfig --list
++ awk '{print $1}'
++ grep 3:on
++ egrep -v 'crond|sshd|network|rsyslog|sysstat'
+ services=

    注：windows换行符是\r\n ,而linux换行符是\n。这样的话，win机器上编写的脚本上传到Linux执行后会报错。

    解决方法1：yum install -y dos2unix && dos2unix win.sh

    解决方法2：将nodepad++转换为Unix格式
