Cloudera_Manager-CDH5.2.0部署

2017-07-10

hadoop 体系衍生了许多商业发行版，目前Hadoop比较流行的主要有2个版本，Apache和Cloudera版本。

Apache Hadoop：维护人员比较多，更新频率比较快，但是稳定性比较差。
Cloudera Hadoop（CDH）：Cloudera公司的发行版本，基于Apache Hadoop的二次开发，优化了组件兼容和交互接口、简化安装配置、增加Cloudera兼容特性。

下载链接：Download Cloudera Manager

整个安装过程，大致上可以分成两个步骤：首先是先安装 CM 控制台（即一个web管理界面），然后通过 CM 来安装 CDH。也就是说，整个CDH集群的安装，是可以在CM界面上进行操作的，包括后面添加节点，也是在CM界面上进行操作。

搭建过程

系统环境

系统：CentOS6.6

Cloudera: CM5.2.0 +CDH5.2.0

内存：主节点：16G，其他slave节点内存8G；CPU均为8核

节点角色说明

节点角色说明
IP 主机名 角色描述
10.1.2.20 cm-master master CM主控节点
10.1.2.21 cm-slave1 Slave1 受控节点
10.1.2.22 cm-slave2 Slave2 受控节点

准备工作(所有节点操作)

1. 修改hostname
   vim /etc/sysconfig/network

2. 关闭SELinux
   getenforce    查看SELinux状态
   vi /etc/selinux/config    若SELinux没有关闭，修改SELinux=disabled.（重启生效）
   setenforce 0   立即生效

3. 关闭防火墙
   service iptables stop
   chkconfig iptables off
   chkconfig iptables --list

4. 网络配置（略）
   设置静态ip以及制定DNS

5. 修改hosts文件，添加如下：

   # CDH Cluster
   10.1.2.20   cm-master
   10.1.2.21   cm-slave1
   10.1.2.22   cm-slave2

6. 重启

安装mysql

略

安装jdk（所有节点操作）

(1) 创建java目录
    mkdir /usr/java && cd /usr/java   

(2)下载解压
   tar -zxvf jdk-7u79-linux-x64.tar.gz  

(3)配置java环境变量
 vim /etc/profile
    #set java environment
    JAVA_HOME=/usr/java/jdk1.7.0_79
    JRE_HOME=/usr/java/jdk1.7.0_79/jre
    CLASS_PATH=.:$JAVA_HOME/lib/dt.jar:$JAVA_HOME/lib/tools.jar:$JRE_HOME/lib
    PATH=$PATH:$JAVA_HOME/bin:$JRE_HOME/bin
    export JAVA_HOME JRE_HOME CLASS_PATH PATH
source /etc/profile

(4)验证
   [root@localhost java]# java -version
   java version "1.7.0_79"
   Java(TM) SE Runtime Environment (build 1.7.0_79-b15)
   Java HotSpot(TM) 64-Bit Server VM (build 24.79-b02, mixed mode)

部署内网ntp时间服务器

参考：NTP服务器部署

下载文件

  • 下载：cloudera-manager.repo

    修改baseurl中cm版本为实际版本，本例中改成5.2.0

    [cloudera-manager]
# Packages for Cloudera Manager, Version 5, on RedHat or CentOS 6 x86_64                  
name=Cloudera Manager
baseurl=https://archive.cloudera.com/cm5/redhat/6/x86_64/cm/5.2.0/
gpgkey =https://archive.cloudera.com/cm5/redhat/6/x86_64/cm/RPM-GPG-KEY-cloudera    
gpgcheck = 1

  • 下载：CM installer

  • 下载：CM rpms，目录中全部下载下来

  • 下载：CDH Parcel 文件，只需要下载如下三个

    CDH-5.2.0-1.cdh5.2.0.p0.36-el6.parcel
CDH-5.2.0-1.cdh5.2.0.p0.36-el6.parcel.sha1
manifest.json

注: 将.sha1文件后缀更改为.sha，同时把内容只保留hash码部分。
(5.2.0版本sha文件内容本身就只包含hash码，不需要修改，其他版本请自行查看！)

Cloudera Manager安装

# 以下操作在所有节点运行
(1)将cloudera-manager.repo文件拷贝到所有节点的/etc/yum.repos.d/文件夹下
   验证repo文件是否起效
   yum list|grep cloudera

  如果列出的不是你安装的版本，执行下面命令重试
   yum clean all 
   yum list | grep cloudera

(2) 将之前下载的rpms文件拷贝到所有节点的/opt目录下
    cd /opt/rpms
    yum -y install *.rpm

(3) 将之前下载的Parcel那3个文件拷贝到/opt/cloudera/parcel-repo目录下（没有此目录则创建）
    cd /opt/cloudera/parcel-repo && ls
        CDH-5.2.0-1.cdh5.2.0.p0.36-el6.parcel
        mainfest.json
        CDH-5.2.0-1.cdh5.2.0.p0.36-el6.parcel.sha
# 以下步骤只在主节点上执行：
(1) 将下载好的bin文件拷贝到主节点任意目录下(我的目录是这个 /opt/cloudera/parcel-repo)
     cd /opt/cloudera/parcel-repo/

(2) 进入该目录，给bin文件赋予可执行权限
    chmod +x ./cloudera-manager-installer.bin

(3)删除db.properties(不删除会导致下一步出错)
    rm -rf /etc/cloudera-scm-server/db.properties

(4)安装CM
   ./cloudera-manager-installer.bin

接下来全部next和yes就行了，因为所需要的rpm包已经提前下载下来了，这个过程会进行的非常快。

CDH集群安装（Parcel方式）

CM安装成功后, 浏览器输入http://ip:7180 (ip是CM安装的主机ip或者主机名) ，用户名和密码都输入admin，登陆进入web管理界面。

注：如果打不开改网页，请耐心等待会。这个服务启动是需要一定时间......

Web初始化配置

(1) 选择部署版本
    免费版即可，继续

(2) 为 CDH 群集安装指定主机。
    cm-master
    cm-slave1
    cm-slave2
    点击搜索，继续

    当然，也可以使用如下方式搜索系列主机，如：
    192.168.1.[101-105]
    slave[1-3].com.cn

(3) 选择存储库
    这里我们采用Parcel离线安装的方式，安装简便、完全离线而且更便于集群的管理。
    然后选择我们之前上传到服务器的CDH5.2.0版本，其他选择无。
    继续下一步

(4) JDK安装选项
    JDK我们在最开始已经安装，所以这一步“取消勾选”，继续。

(5) 提供SSH登陆凭据
    由于CDH会自动管理所有主机间的SSH通讯，所以我们之前并没有手动配置各个节点间的SSH免密登录。在这里统一设置就行了，设置好密码点继续。
    注：每台机器的密码需要一致，且账户具有管理员权限，最好是root。

(6) 群集安装
    这一步错误：socket.gaierror: [Errno -2] Name or service not known
    解决方案：mv /usr/bin/host /usr/bin/host.bak

(7) 安装选定的Parcel
    这一步不需要干预，CDH安装包已经下载，耐心等待部署即可。类似于解压后分发到各节点上。

(8) 检查主机正确性
    警告：Cloudera 建议将 /proc/sys/vm/swappiness 设置为 0。当前设置为 60。使用 sysctl 命令在运行时更改该设置并编辑 /etc/sysctl.conf 以在重启后保存该设置。您可以继续进行安装，但可能会遇到问题，Cloudera Manager 报告您的主机由于交换运行状况不佳。以下主机受到影响。
    解决方法：在会受到影响的主机上执行echo 0 > /proc/sys/vm/swappiness命令即可解决，然后重新检查。

(9) 安装CDH5服务
    CDH5提供了你可能需要的各种服务，需要什么启动什么就行了。这里我选择了自定义，添加HBase，HDFS，MapReduce和ZooKeeper，这里大家按照自己的需求选。

(10) 数据库设置
     CDH默认内嵌了一个数据库，如果你需要MySQL或者Oracle请选择使用自定义数据库，点击测试连接，成功后点继续。
     为测试，暂且使用嵌入式数据库

注意，这里安装Hive的时候可能会报错，因为我们使用了MySql作为hive的元数据存储，hive默认没有带mysql的驱动，通过以下命令拷贝一个就行了：
cp /opt/cm-5.3.4/share/cmf/lib/mysql-connector-java-5.1.25-bin.jar 
/opt/cloudera/parcels/CDH-5.3.4-1.cdh5.3.4.p0.12/lib/hive/lib/

(11) 审核更改
     保持默认即可，继续

(12) 机器设置
     等待Hadoop等各项服务自动部署完成，然后随即进入管理页面


Web管理: http://cm-ip:7180
